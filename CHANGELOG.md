## [Unreleased]

### Fixed

- **UpSet ŌĆö column identifiers (`I1`, `I2`, ŌĆ”) tilt 90┬░ once there are more than 10 intersections.** Horizontal labels collided and became unreadable past ~10 columns because the matrix cell width shrinks as the number of intersections grows. Above the threshold, each label rotates to read bottom-to-top (`transform="rotate(-90 cx y)"`, `textAnchor="end"`, `dominantBaseline="middle"`), occupying the same horizontal width as a single glyph regardless of how many digits the id needs. The bottom lane height and the legend Y-offset now derive from the longest rotated label so the `I# = intersection id ŌĆ”` caption and the SVG canvas (`VH`) grow to accommodate it without clipping. Behaviour is unchanged at Ōēż 10 columns.
- **Venn ŌĆö top "Plot" nav tab now respects pending set selection in Configure step.** Same bug pattern as the UpSet fix below: the `StepNavBar` "Plot" tab routed via `shell.setStep("plot")` directly, bypassing the `onCommit` path the bottom "Plot ŌåÆ" button used. Configure-step checkbox edits were silently discarded and the plot rendered against the previously-committed selection. `canNavigate("plot")` also reported readiness against the committed `setNames` rather than the pending one. Fixed with the same approach: a `navigateStep` wrapper commits `pendingSelection` (when it differs and has 2ŌĆō3 sets, matching Venn's set-count gate) before calling `setStep`, and `canNavigate` now gates on `pendingSelection` while the user is in Configure. Audited the other four plot tools (Lineplot, Scatter, Heatmap, Aequorin, Boxplot) ŌĆö none have the pattern: their step components write edits directly to App-level state via prop setters, so top-nav clicks don't discard anything.
- **UpSet ŌĆö top "Plot" nav tab now respects pending set selection in Configure step.** Previously, the `StepNavBar` "Plot" tab routed via `shell.setStep("plot")` directly, bypassing the `onCommit` path that the bottom "Plot ŌåÆ" button used ŌĆö so if the user toggled checkboxes in Configure and then clicked the top nav, the changes were silently discarded and the plot rendered against the previously-committed selection. `canNavigate("plot")` also reported readiness against the committed `displaySetNames`, not the pending one, so its enabled/disabled state was stale too. Added a `navigateStep` wrapper that commits `pendingSelection` when leaving Configure for Plot (only if it differs from the committed selection and has Ōēź 2 sets), and updated `canNavigate` to gate on `pendingSelection` while the user is in the Configure step. The bottom "Plot ŌåÆ" button behaviour is unchanged.

### Added

- **Venn ŌĆö circle outline toggle in the Display sidebar** ŌĆö the area-proportional Venn circles are drawn with both a semi-transparent fill (`fillOpacity` slider) and a 2 px coloured stroke at `strokeOpacity 0.6`. Added a new `Circle outline` On/Off segmented control below the `Fill opacity` slider, mirroring the `Proportional areas` toggle. Defaults to On so existing plots look unchanged; flipping to Off sets each set-circle's `stroke="none"` and `strokeWidth=0` (the legend swatches keep their outline so the colour dots remain visible against a white background). - **Heatmap ŌĆö invert palette toggle** ŌĆö colour-scale sidebar gained a `Direction` segmented control (`Normal` / `Inverted`) below the `PaletteStrip` preview.
