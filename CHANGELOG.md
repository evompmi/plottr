## [Unreleased]

### Added

- **Venn ‚Äî files with more than 3 columns are now accepted, with a column picker in Configure** ‚Äî previously `doParse` hard-rejected any file whose header row resolved to more than 3 non-empty set columns with a blocking error. Files from upstream pipelines often carry 4+ candidate columns (e.g. "genes up in stress_A / stress_B / control / qc_flag"), and users had to manually prune the file before loading. The tool now parses every column regardless of count and stores them as `allColumnNames` / `allColumnSets`; the Configure step gains a checkbox grid (one toggle per parsed column, each labelled with the column's item-count) that caps selection at 3 and requires at least 2 before the **Plot ‚Üí** button enables. For files with 2‚Äì3 columns the picker is skipped entirely ‚Äî `doParse` auto-commits the full column set and routes straight to the Plot step, so existing single- or two- or three-column workflows see zero UI change. The Plot step keeps its existing per-set toggle (the `activeSets` visibility filter) as a separate axis ‚Äî "which of the file's columns to treat as sets" (committed in Configure) vs. "which of the committed sets to display right now" (toggled in Plot's sidebar). Navigating back to Configure after committing shows the prior selection pre-ticked so re-picking is a quick edit, not a restart; committing a different selection resets `selectedMask` and colour assignments.

- **All plot tools ‚Äî user visual preferences persist across sessions, plus a portable Save / Load preset file** ‚Äî the six plot tools (boxplot, aequorin, lineplot, scatter, venn, heatmap) now remember the user's render-side tweaks (colors, opacities, widths, sizes, toggles, palettes, etc.) between page loads. The page header gains a compact **Settings** strip on the right (same row as the tool title ‚Äî kept out of the sidebar where control real estate is expensive) with three small buttons ‚Äî **Save** (downloads `<tool>-settings.json` ‚Äî a portable JSON snapshot of the full render state, title + axis labels included), **Load** (opens a file picker and applies the settings to the current plot), **Reset** (restores tool defaults and clears the stored preferences). The auto-persist layer deliberately _excludes_ plot/axis labels (`plotTitle`, `plotSubtitle`, `xAxisLabel` / `yAxisLabel` ‚Äî auto-detected via a `/Title|Subtitle|AxisLabel$/` regex, not a hand-maintained allowlist), because those are dataset-specific and silently re-filling a title from the _previous_ experiment is more annoying than useful ‚Äî labels still round-trip through the exported JSON file when the user explicitly opts in. Validation is defensive: tool-name mismatch on an imported file rejects with a one-line inline error, unknown keys are silently dropped, type-mismatched values fall back to defaults (so future rename PRs can't break older files). Storage lives under `dataviz-prefs-<toolName>` in `localStorage`, mirroring the existing `dataviz-theme` convention, with a 300 ms debounce so slider drags don't thrash storage. New shared module `tools/shared-prefs.js` (plain JS, loaded as a global `<script>` like the other `shared-*.js` files) exposes `loadAutoPrefs` / `saveAutoPrefs` / `exportPrefsFile` / `importPrefsFile` primitives plus a compact `PrefsPanel` React component; `PageHeader` (in `tools/shared-ui.js`) gained an optional `right` slot to host it alongside the title. 15 new unit tests in `tests/prefs.test.js` cover the validation / rejection / round-trip paths.

- **Heatmap ‚Äî cell tooltips, interactive selection, and a linked detail panel** ‚Äî hovering any cell in the main heatmap now shows a floating tooltip with the row label, column label, raw value, and (when normalisation is active) the plotted/normalised value. Three ways to build a selection: (1) drag a rectangular brush on the main grid; (2) click any internal branch of a row or column dendrogram to select that subtree's leaves; (3) in k-means mode, click a segment on the cluster-assignment strip to select every row/column sharing that cluster id. Selections are drawn as a white spotlight mask (60 % opacity on every non-selected cell) so the selected rows / columns stand out without any black outlines bleeding into the SVG export, and drive a second `HeatmapChart` rendered below the main one, showing only the selected rows √ó columns with its own SVG / PNG / CSV download buttons and a data table of the raw values (capped at 50 visible rows; CSV always has the full selection). A `Clear` button appears in the top-right of both plots when a selection is active. Selection indices point into the original unordered matrix, so re-clustering doesn't silently mis-highlight ‚Äî any change to mode / k / distance / linkage / normalisation auto-clears the selection. The detail chart itself isn't interactive (tooltips only) to keep selection state single-sourced in the main view. Exporting the detail SVG gives a touch-up-ready figure of just the subset without having to go back to the source data and filter it by hand. The detail plot locks its column width and colour-bar length to the main plot and mirrors the main's dendrogram left-margin, so the two SVGs centre at the same width and the detail cells sit directly under the same columns in the main plot; cell _height_ in the detail is enlarged (14‚Äì28 px, scaled to the detail's own row count) so the selection actually reads as a zoom and row labels stop colliding when the names toggle is on.

- **Benchmark ‚Äî clustering primitives cross-validated against R** ‚Äî the R vs Toolbox comparison suite now exercises `pairwiseDistance` (Euclidean, Manhattan, 1 ‚àí Pearson) and `hclust` merge-heights (5 distance √ó linkage combinations) on a fixed-seed 100 √ó 15 Gaussian matrix. Comparison is done on sorted distance/height vectors so tie-handling differences don't spuriously fail the check while real numerical bugs still stand out. Currently: 8/8 new cases pass, max |Œî| ‚âà 8e-7 ‚Äî well inside the 5e-3 tolerance. K-means is intentionally left out of the benchmark: R's default (Hartigan-Wong, non-k-means++ init, incompatible RNG state) would make any labelled cross-check flaky noise; the deterministic-seed unit tests cover correctness on well-posed inputs.

- **Heatmap ‚Äî visible gaps between k-means clusters** ‚Äî when k-means is active on an axis, cells belonging to different clusters are now separated by a 10 px blank band (the cluster-assignment strip breaks at the same positions). Previously adjacent clusters read as one continuous colour strip with only the fill hue to signal the boundary; the new gap makes the k groups legible at a glance and matches the visual convention established by row/column dividers in tools like Morpheus and pheatmap's `gaps_row`/`gaps_col`. Gaps integrate with `plotW`/`plotH` so the colour bar, row labels, and selection overlays sit past the gapped grid's right / bottom edge; pointer hit-testing reports no hit when the cursor falls in an inter-cluster gap.

- **Heatmap ‚Äî show row / column names toggles, and a built-in 500-row example** ‚Äî tall matrices (hundreds of rows at a 2 px cell-height floor) left no vertical room for per-row labels, so row / column name visibility is now user-controllable via a pair of `Off / On` segmented buttons styled to match the existing clustering and grid toggles elsewhere in the app. Row names default to off (they can't be rendered legibly below ~12 px cells anyway); column names default to on. When labels are hidden the corresponding axis margin collapses to 0 so the plot doesn't reserve empty space. The app now ships with a deterministic 500-gene √ó 6-sample example (5 latent expression patterns √ó 100 genes each, `seededRandom(42)` so it's the same file on every reload) so the zoom / selection / clustering path can be exercised on a realistically tall matrix without having to hand-author one.

- **Heatmap ‚Äî `NumberInput` stepper for k-means k and seed** ‚Äî the two k-means numeric fields (per-axis `k`, 2‚Äì10; RNG seed, ‚â• 1) now render through the shared `NumberInput` component, matching the +/- hold-to-repeat steppers used in Power, Aequorin, and Line Plot. Replaces the bare `<input type="number">` fields, which had inconsistent spacing with the segmented controls above them and no touch-friendly stepper buttons.

- **Heatmap ‚Äî k-means (non-hierarchical) clustering** ‚Äî the row and column clustering controls are now three-state (None / Hierarchical / K-means) instead of on-off booleans. Picking K-means on either axis runs k-means++-seeded k-means (with 8 restarts, inertia-minimising selection) on the normalised matrix (rows as observations, or the transpose for columns), reorders that axis so same-cluster rows/columns sit together (within a cluster, ordered by distance to centroid), and renders a thin Okabe-Ito-coloured cluster-assignment strip in place of the dendrogram. Per-axis `k` is exposed as a 2‚Äì10 spinner (default 3); a separate K-means seed input makes the clustering deterministic and lets users explore alternative initialisations. Hierarchical distance/linkage controls only surface when at least one axis is in Hierarchical mode; the K-means seed only surfaces when at least one axis is in K-means mode. The `.R` script export emits a `set.seed()` + `kmeans()` call (with `nstart = 8`) that reorders the matrix before handing it to `pheatmap`, and attaches a `factor(cluster)` annotation track on the k-means axis so the reproduced figure mirrors the browser one. New shared primitive `kmeans(matrix, k, { seed, maxIter, restarts })` in `tools/stats.js` ‚Äî self-contained (local Park-Miller RNG, no cross-file dependency) so stats.js can be unit-tested in isolation. Handles NaN entries pairwise per feature, falls back gracefully on empty clusters by reseeding to the farthest row.

- **New Heatmap tool** ‚Äî a browser-only tool for visualising numeric matrices (gene √ó sample expression tables, time √ó treatment readouts, metabolite √ó condition panels). Accepts wide-format CSV/TSV with a label column and label row; the numeric interior renders as a coloured grid. Optional hierarchical clustering can be enabled independently on rows and/or columns with three distance metrics (Euclidean, Manhattan, 1 ‚àí Pearson correlation) and three linkage methods (average / UPGMA, complete, single); clustered axes reorder the matrix and grow a dendrogram on top (columns) or the left (rows). Normalisation modes: none, Z-score by row, Z-score by column, log‚ÇÇ(x + 1). When a Z-score mode is active the tool auto-suggests the RdBu diverging palette anchored at 0; the default sequential palette is viridis. Twelve palettes total (viridis / plasma / magma / inferno / cividis / RdBu / bwr / RdYlBu / reds / blues / greens / spectral) ‚Äî these also become available to Scatter via the shared module. **Borderless cells are the default** per a user preference for flat, gapless grids, with integer pixel-snapping and `shape-rendering="crispEdges"` so adjacent cells share a boundary and PNG exports don't open sub-pixel seams; a "No cell borders" checkbox reveals a `ColorInput` and width slider (0.25 ‚Äì 2 px) when unchecked, for readers who want a gridded `pheatmap`-style look. Exports the named SVG groups (`cells`, `row-dendrogram`, `col-dendrogram`, `colorbar`, `row-labels`, `col-labels`, `title`, etc.) expected by Inkscape touch-up workflows, plus a CSV of the plotted values and a runnable `.R` script that reproduces the same clustering, normalisation, and colour scale via `pheatmap` (the raw matrix is embedded so reviewers can regenerate the figure without the original CSV). New shared primitives (`parseWideMatrix`, `pairwiseDistance`, `hclust`, `dendrogramLayout`, `COLOR_PALETTES`, `DIVERGING_PALETTES`, `interpolateColor`) live in `tools/shared.js` and `tools/stats.js` so future matrix tools can reuse them; `scatter.tsx` dropped its inline palette copies and now imports from the shared module, gaining the four new palettes for free.

### Changed

- **Control-panel disclosure sections auto-scroll the sidebar to reveal both the expanded content AND the next section's header** ‚Äî opening a collapsible section in a tool's right-hand control panel (Columns / Groups / Error bars / Axes / Labels / Style / Shape & fill / Data points / Split by / Filters / etc.) used to require a click-then-scroll two-step: the section's new content appeared below the viewport fold of the sticky sidebar and users had to scroll the panel themselves to see it. Now the section root registers a `useEffect` on its open state and, on the next animation frame, calls a new `scrollIntoViewWithinAncestor(el, pad, extraBottom)` helper (in `tools/shared-ui.js`) that walks up to the nearest ancestor with `overflow-y: auto | scroll`, computes how far the container needs to scroll so `el.bottom + extraBottom` lands inside the visible area, and calls `scrollBy({ behavior: "smooth" })` ‚Äî so only the sidebar scrolls, never the surrounding page. The caller passes the measured `offsetHeight` of the next sibling's first child (its disclosure header) as `extraBottom`, so the scroll lands with the current section fully visible PLUS the next section's title strip peeking in ‚Äî preserving context about what's below without hiding it. The helper clamps the delta so the section's own header never scrolls out of view (important for sections taller than the viewport), and is a no-op if the sidebar doesn't overflow or there is no next sibling. Wired up in every `ControlSection` component across `boxplot.tsx`, `lineplot.tsx`, `aequorin.tsx`, plus the inline Filters disclosure in `scatter.tsx`. Heatmap and Venn have no disclosure sections in their sidebars and are unaffected.

- **Disclosure indicator redesigned: neutral circle, crisp SVG chevron** ‚Äî `.dv-disclosure` (the collapsible marker used by every accordion section: StatsTile, Scatter's Filters panel, Aequorin's chart / inset / replicate-table collapse buttons, Line Plot's stats tile, Boxplot's stats tile) had two problems: it was a bright `--cta-primary-bg` accent-blue circle that fought with the surrounding chrome for attention, and it painted its caret by setting `content: ">"` on the `::before` pseudo-element ‚Äî the browser rendered the font stack's fallback ">" glyph, which on macOS picks a skinny, top-heavy character sitting slightly off-centre and read as "cheap" against the rest of the UI. Two fixes applied together. (1) Recoloured the circle with the neutral pair `--neutral-bg` / `--neutral-text` (soft grey-on-grey in light mode `#e5e7eb` + `#374151`, dark-grey-on-light-grey in dark mode `#2e2f3a` + `#d1d5db`) so the marker sits calmly inside panel chrome in both themes rather than pulling focus; a faint `--border` hover state replaces the old accent for affordance. (2) Replaced the text glyph with an inline SVG chevron applied via `mask-image` (`stroke-linecap: round`, `stroke-width: 2.5`, 24√ó24 viewBox, painted through `background-color: currentColor` so it inherits the new `--neutral-text`). The mask approach keeps the marker a single `.dv-disclosure` class with zero React-side changes ‚Äî every existing call-site across `boxplot.tsx`, `lineplot.tsx`, `aequorin.tsx`, `scatter.tsx`, and `shared-stats-tile.js` picks it up automatically. Rotate-on-open behaviour retained; transition bumped from `180ms ease` to `200ms cubic-bezier(0.4, 0, 0.2, 1)` (Material "standard" easing) for a slightly more deliberate feel.

- **SVG export ‚Äî id grouping consistency pass across all plot tools** ‚Äî exported SVGs are routinely re-opened in Inkscape, where named `<g id="...">` groups make individual chart parts (axes, data series, annotations) selectable from the Objects panel. An audit found two structural inconsistencies and a documentation gap: (1) Scatter wrapped its full-canvas and inner-area background fills as bare `<rect id="background">` and `<rect id="plot-area-background">`, both with the same `plotBg` fill ‚Äî they are now wrapped in `<g id="background">` / `<g id="plot-area-background">` groups so they sit at parity with Heatmap (the most recently added tool, used as the structural reference); (2) Heatmap's plot subtitle text was nested inside the `<g id="title">` group, making it impossible to recolour or reposition the subtitle in Inkscape without grabbing the title too ‚Äî promoted to its own sibling `<g id="subtitle">` group, matching Aequorin and Lineplot. Also expanded `CLAUDE.md`'s "Conventional ids" list to enumerate every group id that actually ships across the six plot tools ‚Äî including the previously-undocumented Heatmap groups (`col-dendrogram`, `row-dendrogram`, `col-cluster-strip`, `row-cluster-strip`, `colorbar`, `cells`, `row-labels`, `col-labels`, `selection-mask`), Boxplot's `subgroup-separators` / `subgroup-labels`, Lineplot's `error-bars` / `significance-stars`, plus the per-series id patterns each tool uses (`points-${name}`, `errbars-${name}`, `ribbon-${prefix}`, `cell-${row}-${col}`) ‚Äî so future contributors don't reinvent ad-hoc names that fragment the convention further.

- **Aequorin's Pool-by-name / Individual selector promoted out of the Sample-selection popover, and the Sample-selection pill restyled to match** ‚Äî this control decides the _shape_ of every series that follows (pooled means one line per condition with a ribbon; individual means one line per replicate), so it affects the chart, the inset barplot, the stats tile, and every export. It used to be tucked inside the Sample-selection overlay where it only surfaced once the popover was open. It now lives as a top-right pill group next to the Combined / Faceted toggle, labelled `SERIES ¬∑ Pool by name / Individual`. All three groups (`SAMPLES`, `SERIES`, `LAYOUT`) now share the same visual weight: tiny uppercase captions, 12 px font, 600 weight, 1 px accent-coloured border, and the same padding, so they read as three independent axes rather than one dominant button plus two secondary toggles. Accents stay distinct ‚Äî amber (`--accent-warning`) for Samples, purple (`--cta-dna-bg`, matching the DNA/molarity token family) for Series, blue (`--step-active-bg`) for Layout. Sample selection keeps its sticky scroll-tracking behavior; Series and Layout still scroll away with the chart. No behavior change; `handlePoolChange` is the same callback that used to sit in the popover.

## [2.2.0] - 2026-04-18

### Changed

- **Line Plot's X / Y min & max now round to 2 decimals and step by 0.1** ‚Äî the auto-computed axis bounds that pre-fill all four axis inputs were rounded to 4 decimals, which produced noisy readouts like `0.3724` for typical physiological data. Three changes applied symmetrically to X min / X max / Y min / Y max: (1) the auto path now rounds to 2 decimals; (2) the onChange handler also rounds to 2 decimals so typing, pasting, or stepping can never leak float artefacts like `0.30000000000000004` into state; (3) the +/‚àí buttons now step by `0.1` (was `1`) so adjusting the axis walks the first decimal instead of jumping whole units ‚Äî a typical lineplot range is a fraction of a unit, so stepping by 1 always over-shoots the visible data.

### Fixed

- **Error-bar whiskers on bar charts no longer extend past the axis frame** ‚Äî in Group Plot (bar style) and Aequorin (inset barplot) auto-y mode, when the lower whisker (`mean ‚àí err`) dropped below the baseline (typically 0), the line kept drawing down into the x-axis tick labels. Two changes: (1) Group Plot's auto-y calculation no longer extends `yMin` to include `mean ‚àí err`; it only extends for actually-negative means or data points, so bars stay anchored at 0 by default (matches Aequorin's existing behavior). (2) Both tools now clip the whisker to `[yMin, yMax]` in data-space before projecting to pixels, and drop the end-cap on a clipped end (matplotlib / `coord_cartesian` convention) so the truncation is visible ‚Äî the line remains but the cap disappears, signalling that the error extends beyond the plot range.

### Added

- **Error-bar type called out in the on-plot stats summary for barplots** ‚Äî when "Print summary below plot" is on and the plot is a barplot, the summary gains a fifth line reading `Error bars: None / SD / SEM / 95% CI` that mirrors the current error-bar selector. Applies to Group Plot in bar style (across flat, subgroup, and facet modes) and to Aequorin's inset barplot (which is always a bar chart, so it always shows). Other Group Plot styles (box / violin / raincloud) don't have an error-bar selector, so the line is suppressed there to avoid implying one.
- **"None" error-bar option on Group Plot barplot** ‚Äî the Group Plot (bar style) error-bar selector now reads `None / SEM / SD / 95% CI`, matching Aequorin's inset barplot. Picking "None" suppresses the three whisker lines, drops the error-bar stroke-width slider (nothing to style), and removes the error term from each bar's aria-label. The auto y-axis path treats the error magnitude as `0` in this case so bars no longer reserve headroom for whiskers that won't render.
- **SEM and 95% CI columns in Groups tables, TXT exports, and R-script descriptive stats** ‚Äî the expanded Groups table in every stats tile (Group Plot, Aequorin, Line Plot) now shows `SEM` and `95% CI` alongside `Mean` and `SD`; the half-width is `tinv(0.975, n ‚àí 1) ¬∑ SEM` and displays as `‚Äî` when `n ‚â§ 1`. The ‚¨á TXT export for each tool appends `SEM=‚Ä¶ , 95% CI=‚Ä¶` to every group's descriptive line so the downloaded summary matches what's on screen.
