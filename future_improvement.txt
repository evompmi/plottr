High impact, moderate effort                          

  1. PNG export — Scientists need raster images for papers/posters. You could use <canvas> 
  to convert the existing SVG without any server. This is probably the single most-requested
   missing feature in any SVG-based dataviz tool.
  2. Accessibility basics — The SVG charts have zero ARIA attributes (<title>, <desc>,      
  role="img"), and form inputs lack associated <label> elements. A single pass through each
  tool would make a big difference.                                                         
  3. Log-scale axes — Very common need in plant science (dose-response curves, growth     
  kinetics). Currently no log-scale option on any tool.

  Medium impact, low effort

  4. Axis min/max controls for bar/box — Scatter already has these; the other chart tools
  auto-compute ranges with no override. Scientists often need to match axis ranges across
  figures.
  5. Copy SVG to clipboard — Aequorin has copy-CSV, but no tool lets you paste a chart
  directly into a slide deck. A few lines with the Clipboard API.
  6. Legend position control — Legends are always at the bottom. A
  top/bottom/left/right/none selector would help with layout.

  Code health

  7. Boxplot JSX readability — boxplot.jsx is written in an extremely dense, nearly minified
   style (the entire BoxplotChart is ~35 lines of run-on code). The other three tools are
  well-formatted. Reformatting boxplot would make it much easier to maintain.
  8. Component test coverage — You have 339 tests but they all cover shared.js utilities.
  Zero tests touch React components. Even lightweight render-smoke tests for the chart
  components would catch regressions.
  9. Responsive chart sizing — All charts use fixed viewBox dimensions. They work on desktop
   but can be unreadable on mobile. A ResizeObserver-based wrapper or percentage-based
  viewBox would fix this.

  Nice-to-have

  10. Dark mode — Hardcoded light colors throughout. Not urgent but increasingly expected.
  11. Undo/redo for column rename/reorder/filter steps.
  12. Confidence intervals on bar chart error bars (currently only SEM/SD).

  If I had to pick one thing to build next, it'd be PNG export — it's the feature gap users
  will hit most often and it's self-contained.
