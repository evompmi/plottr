  Medium Priority

  4. var vs const/let inconsistency
  shared.js and shared-components.js mix var and const throughout. Since these are plain JS (no build step),
  var hoisting can cause subtle scoping bugs. Converting to const/let is a straightforward cleanup.
  Files: shared.js (lines 43-45 toolIcon, lines 305-334 downloadPng), shared-components.js (throughout).

  5. O(n^2) legend block positioning (shared-components.js:122-135)
  blocks.slice(0, bi).reduce(...) runs inside a loop — quadratic slicing for every block. Should accumulate
  offset in a single pass. Low real-world impact (legend blocks are typically < 5), but easy to fix.

  6. Stale callback refs in faceted charts (bargraph.jsx, boxplot.jsx)
  ref={el => { facetRefs.current[fd.category] = el; }} creates a new function per render per facet.
  React calls the old callback with null then the new one with the element on every render. This is inherent
  to dynamic refs inside .map() — the only true fix is extracting each facet item into a memoized component.
  Low practical impact (facet count is small, DOM work is minimal).

  7. SVG accessibility
  All chart elements (<circle>, <rect>, <path>) lack aria-label or role="img". Adding:
  - role="img" and a <title>/<desc> to root <svg> elements
  - aria-label on data-bearing elements (bars, points, boxes)
  Would make charts screen-reader interpretable. Medium effort across all chart components.

  Low Priority

  8. Duplicated margin constants — bargraph.jsx, scatter.jsx, boxplot.jsx each define their own margin
  objects. Could consolidate in shared.js, but each tool has different bottom-margin logic (angle-dependent),
  so a shared constant would need per-tool overrides anyway. Marginal benefit.

  9. Outdated Math.max.apply(null, arr) pattern (shared-components.js) — replace with Math.max(...arr).
  Pure style cleanup, functionally identical.

  10. Color input regex (shared-components.js:7) — Only accepts 6-digit hex (#aabbcc), rejects valid
  3-digit hex (#abc). HTML5 color pickers always produce 6-digit hex, so this is only a problem if users
  type values manually.

  11. No minification in build — esbuild scripts don't pass --minify, so output is larger than necessary
  for production. Add --minify to the build script in package.json for smaller deployments.

  ---
  Not Issues

  - The plain JS constraint on shared.js/shared-components.js is documented and intentional.
  - The seeded PRNG (Park & Miller LCG) is correct for jitter use.
  - No server-side code means no injection/auth risks — data stays client-side.
  - Simple inline handlers on native elements (e => updVis({...})) are not a performance concern —
    useCallback only helps when callbacks are passed to React.memo-wrapped children.
