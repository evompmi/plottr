# Plöttr v1.0.3 — release notes

_Released 2026-05-01._

A naming, screenshots, and copy-refresh release: the aequorin tool gets a
broader user-facing label ("RLU timecourse"), two missing screenshots
(UpSet, Heatmap) land in the README grid, every plot tool's in-app "How to
use" panel resyncs with what actually shipped, and a brand-new
`privacy.html` makes the data-flow story explicit instead of hiding it
behind a tooltip.

## Changed

**Aequorin tool renamed to "RLU timecourse".** The tool always plotted luminescence (RLU) time-courses with optional Ca²⁺ calibration on top, but the name "Aequorin Ca²⁺ Calibration" foregrounded the niche calibration feature over the broader use case. The tool is now "RLU timecourse" everywhere a user sees the name (page `<title>`, landing-page tile, in-app `PageHeader`, How-to card, ErrorBoundary fallback, default download filenames `rlu_timecourse_example.tsv` / `rlu_timecourse_stats.txt`, TXT and R-script report headers, SVG `aria-label`). The aequorin Ca²⁺ calibration is now positioned as a "plus" feature: surfaced in the How-to subtitle (`"Plot luminescence over time • optional aequorin / Ca²⁺ calibration • Σ barplots"`), described once in the purpose paragraph, and the Configure step's dropdown gets an explicit "Aequorin calibration formula" label so the term still anchors the relevant control. Internal identifiers (file paths `tools/aequorin/`, component names `AequorinStatsPanel`, localStorage key `aequorin`, runtime global `__AEQUORIN_EXAMPLE__`, citation comments in `helpers.ts`) are deliberately left as-is so existing users' saved auto-prefs and any external links to `tools/aequorin.html` keep working — the rename is a user-facing label change, not a structural rewrite.

**In-app "How to use" panels resynced with current feature surface.** A drift audit across all seven plot tools found the panels had aged behind the implementation in five of them (lineplot and upset were already accurate). **Aequorin** — Configure-step bullet list no longer mentions Pool/Individual or column on/off (those moved to the plot-page Conditions tile + Sample-selection overlay long ago); Σ-barplot error bars now correctly listed as None / SEM / SD / 95 % CI (panel previously said "SD/SEM"); plot-step bullet now mentions SVG / PNG export instead of CSV-only, and a new Configure-step bullet flags the calibrated-CSV download. **Boxplot** — Workflow list now includes Subgroup by (the per-group splitter added in the audit-23 facet × subgroup change), SVG / PNG export (was SVG-only), and a new bullet covering the entire stats panel — auto-routed test selection (t / Welch / Mann–Whitney / ANOVA / Welch-ANOVA / Kruskal–Wallis), post-hocs (Tukey / Games–Howell / Dunn + BH), CLD letters and significance brackets, plus TXT and runnable R-script reports. **Heatmap** — step 2 expanded from "Z-score by row + UPGMA + diverging palette" to the four-mode normalisation (None / Z row / Z col / log₂), the 3 distance × 3 linkage hierarchical grid plus k-means with seed control, and the dendrogram / cluster-strip visibility toggles; new step 3 covers SVG / PNG / CSV / R-script export and drag-to-zoom sub-region export. **Scatter** — added an "Overlays" card describing the linear regression with R² / p-value label and reference lines (both substantial features that the panel didn't mention at all); palette count corrected from 8 to 12; SVG / PNG / CSV export pill added. **Venn** — upload hint and How-to subtitle now flag long-format support (two columns: item, set) which the tool has auto-detected and reshaped on upload for some time but the docs still claimed wide-format only. No code-path changes; pure copy + JSX edits in each tool's `steps.tsx` / single-file source.

## Added

**Screenshots for UpSet and Heatmap.** `docs/screenshots/` was missing both — the README's screenshot grid covered every other plot tool, so anyone landing on the repo had no visual for the two newer additions. Captured the chart-only SVG export from each tool (matching the style of the existing screenshots — the boxplot one is also a chart export with stats footer, not a full-UI capture), then composited each onto a 1648 × 1250 white canvas with the chart at its natural aspect ratio centred (Inkscape renders to natural-aspect inner PNG, Pillow pastes onto the white canvas). The two-step pipeline matters because the bundled heatmap example is 500 genes — intrinsically portrait — so a 60-gene synthetic dataset is injected via a `DataTransfer`-driven file-input event for the screenshot only; the in-app example dataset is unchanged. README screenshot table extended from 3 × 2 to 4 × 2 — Venn / UpSet on row 3, Heatmap / Power Analysis on row 4. The capture path is automated in `scripts/gen-screenshots.mjs` (selenium-webdriver + system Firefox + geckodriver + Inkscape + Pillow; selenium kept out of `package.json` so a normal `npm install` stays lean — install with `npm install --no-save selenium-webdriver` when re-running the script).

**Privacy / data-flow page (`privacy.html`).** The "Data stays in your browser" trust badge on the landing page used to be a tooltip-only `<span>` — clicking it did nothing. Now it links to a dedicated page mirroring `benchmark.html`'s shell (theme preload, theme toggle, breadcrumb back to `index.html`). The page leads with a single inline-SVG schematic showing a static-file host (GitHub Pages) sending the page in once at load (one-way accent arrow into the trust boundary); a dashed "Your computer" boundary box containing the Plöttr app, your local CSV/TSV input, and a stack-of-documents output icon labelled "SVG · PNG · CSV · R / to your local disk". The Plöttr app box reads simply "Plöttr · parses · computes · renders · entirely in your browser". Underneath the diagram: three plain-language cards — **Your data is safe** ("the chart and the statistics are built right there, inside your browser tab. Plöttr has no servers."), **No monitoring** ("no analytics, no cookies, no trackers..."), **Open to scrutiny** ("read it, fork it, or run a local copy" linking to the GitHub repo). Bottom of the page: an "inspect-it-yourself" block with the `git clone` recipe for fully-offline operation. Theme toggle + cross-page `?theme=` query-param push reuse the existing `data-benchmark-link` plumbing so dark mode persists across navigation. The badge SVG, hover styling, and layout match the existing trust-badge pair on the landing page; this is the first time clicking it leads anywhere.
