# Plöttr v1.9.2 "Fine Print" — release notes

A precision-and-honesty release. The R cross-validation benchmark now holds the
`tools/_core/stats/*` kernel to tolerances calibrated to how well it _actually_
agrees with R — 4 to 8 orders of magnitude tighter than the old blanket bar —
and the report says exactly which numeric space each delta lives in instead of
quietly mixing them. Two UI papercuts go with it: "Send feedback" no longer
dead-ends for people without a mail client, and the chart "Text size" slider
scales every label rather than only the tick values.

The name is the theme: this release is about the fine print — the digits past
the decimal the benchmark used to wave through, now read out loud.

## 🔧 Changed

### Benchmark tolerances calibrated to observed agreement

The R cross-validation suite (`benchmark/run.js`, rendered to `benchmark.html`)
previously passed any statistic within a single **absolute 5e-3** of R's value.
Measuring the real per-metric agreement on the checked-in fixtures showed that
bar was **4–8 orders of magnitude looser** than the truth: the algebraic
statistics (t, F, H, U, r/ρ/τ, Cohen's d/g, …) agree with R to ~5e-11, and even
Shapiro-Wilk W (Royston AS R94) to ~4e-9. A regression that shifted a statistic
by ~1e-3 would have rendered green.

The single bar is now three tiers, each ~20–150× above its measured floor:

| Tier            | Tolerance | Metrics                                                                | Floor / margin |
| --------------- | --------- | ---------------------------------------------------------------------- | -------------- |
| `TOL_STAT`      | **1e-7**  | test statistics & point estimates; Shapiro-Wilk W; Pearson Fisher-z CI | 3.9e-9 / 25×   |
| `TOL_NCT_CI`    | **1e-5**  | noncentral-t effect-size CIs (Cohen's d CI95)                          | 4.7e-7 / 21×   |
| `TOL_QTUKEY_CI` | **1e-3**  | studentized-range CIs (Tukey HSD lwr/upr)                              | 2.87e-5 / 35×  |

The tiers are dictated by the data, not picked round: the Tukey CI bounds ride
the `qtukey` inversion envelope (~3e-5) and the Cohen's d CIs the Cumming–Finch
noncentral-t pivot (~5e-7), so each gets its own bucket rather than dragging the
whole suite down to the loosest member.

The **absolute p-value tolerance** (for p ≥ 0.01) dropped the same way, from
**5e-3 to 1e-5**. The observed floor there is ~8e-7 — the rank-based / post-hoc
adjustments (Dunn, Tukey, Games-Howell pAdj); the parametric p-values themselves
agree to ~1e-12. The old 5e-3 could mask a ~1e-3 p-shift, enough to flip a
decision near α. The deep-tail (p < 0.01) log-space rule is unchanged.

All 524 comparisons still pass, 0 fail. The new tiers turn the public trust
artefact from "passes a bar 10⁴–10⁸× looser than reality" into a check that
would actually go red on a real regression.

### Benchmark report: separate normal-space and log-space delta columns

The report's single **|Δ|** column carried whichever space governed each row —
an absolute difference for most rows, but a **log-ratio** for deep-tail
p-values — so the headline "max |Δ|" silently mixed the two. A 3.16e-2 _log_
value sat in the same column as 5e-11 _absolute_ ones and looked like a
normal-space delta blowing past tolerance, when it was a perfectly fine
log-space agreement.

The comparison helpers now always carry both a normal-space `delta` and a
log-space `logDelta`, and the table renders them as distinct **|Δ|** and
**|Δ ln p|** columns (with "—" where a space doesn't apply, e.g. the log column
on a t-statistic). The summary reports both maxima separately —
`max |Δ| (normal)` and `max |Δ ln p| (log)`.

While in there, the SciPy comparison counts now format with a
locale-independent thousands separator instead of `Number.toLocaleString()`,
whose separator follows the host machine's locale — that had been making
`benchmark.html` drift (`1 083` ↔ `1,083`) on every regeneration. Regenerating
the page now reproduces it byte-for-byte on any machine.

### "Send feedback" opens an in-app dialog

Clicking "Send feedback" used to set `window.location` to a `mailto:` URL. For
anyone without a configured mail client that did nothing visible — and there is
no browser API to detect whether a mail handler exists, so the failure couldn't
be caught and fed back. The click just looked dead.

It now opens a small in-app dialog showing the destination address, the subject,
and the prefilled message, with two always-available paths: **"Open in email
app"** (the `mailto:`, for those who do have a client) and **copy-to-clipboard**
buttons for the address and the message, so a webmail user can paste straight
into Gmail / Outlook. Plöttr still sends nothing automatically. The dialog is
keyboard-accessible: Escape closes it, focus moves into it on open and returns
to the trigger button on close.

### "Text size" slider scales all chart text

The "Tick label size" control (Group Plot, Scatter, Line Plot, RLU, Volcano)
previously sized only the x/y tick values. It now also scales the axis labels,
title, and subtitle proportionally, and is relabelled **"Text size"** (en) /
**"Taille du texte"** (fr). At larger sizes each label's outer edge is pinned —
baseline nudged inward, left margin and title band widened — so the bigger text
no longer crops against the canvas edge. The default (11 px) renders unchanged.
