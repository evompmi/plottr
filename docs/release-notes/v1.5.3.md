# Plöttr v1.5.3 — release notes

_Released 2026-05-16._

A one-fix patch on top of v1.5.2: exported R scripts no longer inline
their data on lines too long for R to parse.

## 🐛 Fixed

**Exported R scripts no longer emit data on lines too long for R to
parse.** Every Plöttr R-script export inlines the plotted data as
`c(...)` literals so the script is self-contained. Two of those paths
emitted the data on a single unbroken line: the scatter correlation
export put each of the `x` and `y` vectors on one line, and the heatmap
export put every row label, every column label, and — for wide matrices
— every matrix row on one line. The bundled 500-gene example heatmap
therefore produced a ~500-element `row_labels <- c(...)` line straight
out of the box.

R's parser rejects an input line longer than 4094 characters, and these
snippets are typically pasted into the RStudio console rather than
`source()`-d from a file — so an over-long single line was silently
truncated mid-literal, producing a syntax error or, worse, a quietly
wrong dataset.

The root cause was four divergent code paths doing the same job: a
private `_wrapC` in `_shell/r-export.ts` (wrapped), a separate private
`wrapC` in `volcano/reports.ts` (wrapped), the exported `formatRVector`
(never wrapped — scatter's path), and heatmap's hand-rolled emit (never
wrapped). They are now consolidated into one exported helper,
**`wrapRItems`**, with a hard **120-character-per-line cap**
(`R_MAX_LINE`). `perLine` remains a soft per-caller hint — heatmap, for
instance, still asks for one matrix row per line — but the 120-character
cap always wins, so a wide matrix or a long label list re-wraps
automatically.

Two smaller cleanups rode along: the factor-level vector that boxplot /
aequorin / lineplot emit is now a standalone `df_levels <- c(...)`
assignment instead of being inlined into a long `factor(...)` call (its
~40-character prefix is invisible to the wrapper), and
`volcano/reports.ts` now imports its R-string helpers from
`_shell/r-export` like every other tool instead of relying on ambient
globals.
