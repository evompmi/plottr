# Pl├╢ttr v1.2.0 тАФ release notes

_Released 2026-05-04._

This release closes a long architectural cycle (every plot tool in its own
folder, full TypeScript `strict: true`, hooks-rules at error level, custom
theme-var lint), ships two cross-cutting user features (R-typical discrete
palette picker on every per-group sidebar; search-by-name for volcano), and
bundles a handful of UX/visual polish fixes that accumulated alongside the
refactor.

The prose below is the long-form context тАФ what changed, why, and how тАФ that
historically lived inside the `CHANGELOG.md` entry. The changelog itself now
keeps one bullet per item with a link back here so the file stays scannable.

## ЁЯПЧ Tool-split refactor тАФ every plot tool in its own folder

Boxplot, aequorin, heatmap, volcano were already folder-shaped after the
helpers-split that landed in v1.0.x. This cycle finishes the job: `venn.tsx`
тЖТ `venn/{index,chart,steps,controls,plot-area,helpers,reports,...}.tsx`,
same shape for `lineplot/`, `scatter/`, `upset/`. All eight plot tools now
follow one canonical layout.

esbuild entrypoints updated to `tools/<tool>/index.tsx`; tsconfig include
broadened to `tools/**/*.ts(x)`. The split surfaced (and the broadened
typecheck caught) three runtime crashes тАФ scatter, lineplot, and upset all
shipped tool-split but were silently passing the old single-star include
glob. Fixed in the same cycle.

The cost of adding a ninth tool is now mostly content тАФ paste the canonical
file layout, fill in the helpers/chart/steps, wire to `tools/_shell/`.

## ЁЯЫб TypeScript strictness тАФ full `strict: true` across the tool surface

Phased in over six commits:

1. Safe flags (`noImplicitReturns`, `noFallthroughCasesInSwitch`,
   `strictFunctionTypes`, `noUnusedLocals` opt-in tier) + R-export surface
   typed.
2. `strictNullChecks` enabled.
3. `noImplicitAny` for the pure-helper tier (`_shell/**`,
   `<tool>/helpers.ts`, `venn/*.ts`).
4. Component prop types tightened for the `shared-*.js` family.
5. `noImplicitAny` enabled globally.
6. `strict: true` (drops the redundant flag list since strict implies
   them all).

Pure helpers are fully annotated. Sprawling step-component prop bags are
permitted to use `: any` on the outer destructure (documented in
CLAUDE.md) тАФ the wiring layer is intentionally soft so React-tier types
don't become a refactor tax. The math layer is rigorous.

The migration caught a long-standing latent bug: a Perl-heredoc shell-escape
during the `noImplicitAny` mass-edit silently rewrote ~24 sites with
literal `$1: any, $2: any` instead of the captured parameter names;
recovered by hand from each tool's pre-migration version.

## ЁЯОи Custom ESLint rule тАФ `plottr/no-chrome-hex-literal`

Defined in `scripts/eslint-rules/no-chrome-hex-literal.js`. Fires on any
inline `style={{ key: "#abc..." }}` JSX outside an SVG subtree, enforcing
the CLAUDE.md theming contract: chrome colours must reference CSS variables
(`var(--тАж)`), while SVG chart internals stay as hex literals so exported
charts render the same way in any reader.

Blocks an entire class of theming-regression bug at lint time. The rule is
deliberately narrow тАФ only inline object expressions тАФ so identifier refs
(`style={someStyle}`) and spreads pass through unchecked, matching existing
codebase patterns.

## ЁЯМИ Discrete-palette picker тАФ all eight plot tools

User-facing feature shared across every per-group / per-category /
per-set / per-condition sidebar.

New shared file `tools/shared-discrete-palette.js` exposes an
**11-palette catalogue**:

| key           | source                           | colour-blind safe |
| ------------- | -------------------------------- | ----------------- |
| `okabe-ito`   | Wong 2011 (default)              | тЬЕ                |
| `tab10`       | Tableau10                        | тЪа                 |
| `set1`        | ColorBrewer                      | тЭМ                |
| `set2`        | ColorBrewer                      | тЪа                 |
| `set3`        | ColorBrewer                      | тЪа                 |
| `dark2`       | ColorBrewer                      | тЬЕ                |
| `paired`      | ColorBrewer                      | тЬЕ                |
| `pastel1`     | ColorBrewer                      | тЪа                 |
| `pastel2`     | ColorBrewer                      | тЪа                 |
| `ggplot2-hue` | `hcl(h, l=65, c=100)` at runtime | тЪа                 |
| `viridis-d`   | even-spaced sample of viridis    | тЬЕ                |

ColorBrewer attribution comment in the file header (Apache 2.0). ЁЯСБ glyph
in the dropdown marks the colour-blind-safe entries.

Per-tool integration:

- **Boxplot** тАФ two independent dropdowns: per-group + per-Color-by point
  category. Each persists its own `vis.discretePalette` /
  `vis.categoryPalette`.
- **Scatter** тАФ drives the discrete colour aesthetic when a categorical
  colour column is mapped. The continuous `colorMapPalette` and the single
  `pointColor` (no-mapping case) are unaffected by design тАФ the discrete
  palette only applies to the categorical-discrete path.
- **Lineplot** тАФ drives per-series line colours via `computeSeries`.
- **Venn** тАФ drives per-set fill colours. `setColors` lifted into
  `vis.setColors` so the palette and hand-edits both round-trip through
  PrefsPanel and auto-prefs (side benefit: now persists across reloads).
- **Aequorin** тАФ drives per-condition colours. `detectConditions` accepts
  a palette name and seeds via `resolveDiscretePalette`.
- **Volcano** тАФ maps the resolved palette into the two significant slots
  only тАФ `[0]` тЖТ up, `[1]` тЖТ down. The non-significant slot stays at its
  default neutral grey regardless of palette so the up/down splay reads as
  the signal; user-edits remain available.

Dropdown chrome uses `className="dv-select"` so it themes correctly in dark
mode. Default `okabe-ito` is byte-identical to the prior `PALETTE`, so
existing data files reload identically.

A self-healing useEffect on volcano keyed on `vis.discretePalette` re-pins
`colorNs` to the canonical grey on every palette change AND once on mount,
which heals two glitches the user surfaced during review:
(a) a brief intermediate build mapped the palette's last hex into ns;
(b) native `<select>` doesn't fire onChange when re-picking the
already-selected value.

Coverage: +17 unit tests covering catalogue shape, sentinel handling
(`ggplot2-hue` and `viridis-d` are runtime-generated), modulo recycling,
ggplot2-hue uniqueness, and the okabe-ito = PALETTE regression guard.

## ЁЯФН Volcano тАФ search points by feature name

User-facing feature: paste a list of gene names, get them highlighted on
the plot.

New _Search by name_ input inside the Labels tile. Type a gene/feature
name (or paste a comma- / newline-separated list) and press Enter / Add тЖТ
matching points get the same black ring + leader line + label as
click-to-label.

UX details:

- Live debounced (150 ms) match-count readout: `"12 matches"` /
  `"no matches"` / `">50 matches тАФ labels may overlap"` /
  `"47 matches ┬╖ 3 unmatched"`.
- An expandable list of unmatched tokens (paste-list typo finder).
- Disabled with hint when no label column is picked.

Match semantics:

- Split on `,` and `\n` тЖТ tokens.
- `trim()` each, drop empties, dedup case-folded.
- Case-insensitive substring per token: a point matches if any token is a
  substring of its lower-cased label.
- Skips points with `label == null`.

Pure helper `matchPointsByLabel(points, query, labelLowerCache?)` lives in
`tools/volcano/helpers.ts` тАФ fully testable, no React, no DOM. Optional
precomputed lower-cased label cache for hot-loop callers (the picker
component memoises it on the `points` reference so a 50 k-point dataset
stays snappy under live preview).

Defensive trim of source labels in `buildPoints` so a sloppy CSV entry
like `"AT1G01010 "` no longer fails an exact-match search.

Zero chart changes тАФ search-added points ride on the existing
`manualSelection: Set<number>` infrastructure via a new `addToManualSelection`
callback alongside `togglePointSelection`. Same Clear button covers both
click- and search-added points; SVG/PNG export captures the labels for
free.

Coverage: +15 unit tests in `tests/volcano.test.js` (case-insensitivity,
comma/newline/mixed splitting, dedup, null-label skip, empty/whitespace,
unmatched-token reporting, trailing-whitespace, duplicate tokens,
defensive non-string input, cache override).
