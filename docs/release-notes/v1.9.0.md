# Plöttr v1.9.0 "Audible" — release notes

The accessibility release. Two charts that a screen reader literally could not
read — the heatmap and the volcano plot, both rendered as a single SVG image
with no per-value DOM — now expose their data as a real, semantic table; every
slider and colour picker announces what it controls instead of a bare "slider";
and the heatmap/volcano image descriptions speak the user's language. Alongside
the a11y work: a path-traversal hole in export filenames is closed, continuous
colormaps flag which ones are colour-blind-safe, a reflexive page reload no
longer silently discards a loaded dataset, and five chart tools gain a tick
label-size slider for more legible exports.

The name is the theme: the charts now speak. What was a mute picture to a
screen-reader user is, in this release, audible.

## 🔒 Security

### Export filenames are sanitised against path traversal (Zip Slip)

Several exports embed a user-supplied label in their filename — an aequorin
condition name, a boxplot facet value, a Venn region label. Nothing stopped
that label from containing `/`, `\`, or `..`, so a maliciously- (or merely
awkwardly-) named condition such as `../../evil` could steer a saved file out
of the folder the user chose, or — once bundled into a multi-file `.zip` — let
an entry escape the archive root when extracted (the classic "Zip Slip").

The fix is one chokepoint in `_core/download.ts`: every save path now strips
path separators, control characters, and leading dots from the user-supplied
portion before it reaches `showSaveFilePicker` / `showDirectoryPicker` / the
`<a download>` fallback. `buildZip` independently re-sanitises every entry name
as defence-in-depth, so even a caller that bypassed the download helper can't
write a traversal path into an archive. Realistic filenames are unaffected; the
sanitiser only collapses the characters that have no legitimate place in a leaf
name.

## ✨ Added

### Keyboard / screen-reader access to heatmap & volcano data

The heatmap and volcano plot are drawn as one `role="img"` SVG with no DOM node
per cell or per point — efficient to render, but completely opaque to a keyboard
or screen-reader user, who had no way to read an individual value. Each chart
now offers a collapsible **"Show values as table"** disclosure beneath it: a
real semantic `<table>` with `scope`-ed headers.

- **Heatmap** — the matrix in display order (respecting any active clustering /
  reordering), row and column headers scoped to their axes, each cell its
  numeric value.
- **Volcano** — one row per point: the point's label, its log₂ fold-change, its
  p-value, and its significance class (up / down / not-significant).

The disclosure is keyboard-reachable and fully translated (EN/FR). For very
large data the table is capped with a visible note explaining the cap, so the
DOM can't blow up on a pathological matrix.

### Colour-blind-safe markers on continuous colormaps

The discrete-palette picker already flagged colour-blind-safe palettes with a
👁 marker. The heatmap and scatter **continuous**-colormap pickers now do the
same, plus a short legend, so a user can see at a glance which schemes are safe
and avoid the one rainbow option (`spectral`) that leans on red–green
discrimination. Defaults are unchanged — viridis, which is already CB-safe.

### Reload guard for in-progress data

Plot data lives only in the browser's memory: there is no server and nothing is
persisted. A reflexive Ctrl-R or an accidental tab-close therefore used to throw
away a pasted/loaded dataset and the entire plot tuned on top of it, with no
warning. The plot tools now arm the browser's native "Leave site?"
`beforeunload` confirmation as soon as a dataset is loaded (past the upload
step) and disarm it otherwise. Navigation _between_ tools inside the SPA is
explicitly exempt — the guard only fires on a genuine reload or close, not on
moving from Boxplot to Volcano.

### Tick label-size control

Group Plot, Scatter, Line Plot, the RLU time-course, and Volcano now expose a
**"Tick label size"** slider (in the style panel for Volcano), scaling the x/y
tick _value_ text from the default 11 px up to 22 px for charts and exports that
need to read at a distance. Two layout details ride along so larger text stays
clean: the x-axis labels nudge downward as the font grows so they clear the axis
line above them, and the left margin widens to fit big y-tick numbers (e.g.
uncalibrated RLU counts) so they never overrun the y-axis label.

## 🐛 Fixed

### Heatmap & volcano chart descriptions now translate

The `role="img"` `<title>` / `<desc>` on both charts, and the per-group
`aria-label`s (matrix dimensions, the row- and column-label groups, the
per-class point counts on the volcano), were hard-coded English — with
English-only pluralisation rules baked in. A French screen-reader user heard the
wrong language, sometimes with the wrong plural form. All of those strings now
come from the i18n catalogues and pluralise through `Intl.PluralRules` in both
locales, so the spoken description matches the rest of the UI.

### Heatmap clustering no longer freezes the tab on huge matrices

Heatmap clustering runs on the main thread (there is no Web Worker), so a
pathological-but-legal CSV — thousands of rows and columns, yet still under the
2 MB ingest limit — could lock the tab for tens of seconds to minutes while the
distance matrix and linkage were computed. Clustering is now **capped per axis,
by mode, matched to each mode's measured cost**:

- **Hierarchical** — O(N³) with an N×N distance matrix — caps at **2,000**
  observations per axis.
- **k-means** — near-linear — caps at **10,000**.

Above the cap the heatmap falls back to file order on that axis and shows a note
explaining why, rather than hanging. The split exists because the two modes have
genuinely different cost curves; a single shared cap would either throttle
k-means needlessly or leave hierarchical free to freeze. Realistic heatmaps are
nowhere near these limits — the bundled demo is 500×6.

### Sliders and colour pickers are now labelled for screen readers

Every range slider (`SliderControl`) now associates its visible label with the
`<input>` via `aria-labelledby`, so it announces e.g. "Point size" instead of a
bare "slider". Every colour picker (`ColorInput`) now takes a `label` and gives
both the swatch button and the hex field an `aria-label` — the group / series /
role name it edits, or a translated "Color" fallback — so a screen-reader user
can tell which series each control affects. These labels are threaded through
the per-group, per-set, and per-condition editors across all eight plot tools.
