# PlГ¶ttr v1.4.1 вЂ” release notes

_Released 2026-05-11._

A patch release built around three concerns: **(a)** every chart export now
carries a small `PlГ¶ttr v<version>` attribution mark in a reserved bottom
band; **(b)** the SPA bundle is now lazy вЂ” first-visit download shrinks by
~60вЂ“95 % depending on route вЂ” with two failure-mode fixes that came out of
shipping the split; **(c)** an accessibility sweep covering chart-level
`role="img"` / `aria-label` / `<title>` / `<desc>` attributes on the two
charts that were missing them (heatmap, volcano), per-element labels on the
four sparse charts (lineplot, aequorin, upset, scatter), site-wide
honouring of `prefers-reduced-motion`, a WCAG 3:1 bump on the volcano "not
significant" default colour, and a new `npm run audit:contrast` script
that locks the palette catalogues against the same bar. Two power
calculator chrome fixes and a small test-loader bug round out the set.

## вњЁ Added

**`PlГ¶ttr v<version>` attribution mark on every chart export.** Every SVG
(and the PNG rasterised from it) now carries a small italic
`PlГ¶ttr v<version>` stamp in the bottom-right of the canvas, sourced from
`window.__APP_VERSION__` so a version bump propagates without any code
change. Implemented as a single shared `appendPlottrAttribution()` helper
in `tools/shared.js`, called from `serializeSvgForExport` on the export
clone only вЂ” the plot area, axes and existing margins stay pixel-identical
to the pre-change export; only the canvas grows downward by 14 px to make
room. The wrapper is `<g id="plottr-attribution"
data-plottr-version="v1.4.1"><text font-size="8" font-style="italic"
fill="#999" text-anchor="end" font-family="system-ui, -apple-system,
sans-serif">PlГ¶ttr v1.4.1</text></g>` exactly вЂ” journals that require
unbranded figures can strip it with one selector. `downloadPng` now sizes
its raster canvas from the watermarked clone's viewBox rather than the
live element so the stamp renders sharp at the configured `scale` (2Г— by
default). The implementation is idempotent вЂ” repeated calls don't stack
bands вЂ” and gracefully no-ops on SVGs without a parseable viewBox or
width/height. 11 new tests in `tests/svg-export.test.js` (happy-dom
environment) pin the spec contract: viewBox grows by exactly 14, the
group/text attribute set matches the spec verbatim, the live SVG is never
mutated, the inline `max-width:100%;height:auto` style is still stripped
(the existing Inkscape workaround), the `shape-rendering="crispEdges"`
heatmap workaround still fires, and the version string falls back to
`v?` when `__APP_VERSION__` is missing rather than throwing.

**`npm run audit:contrast` вЂ” palette contrast audit script.** Walks every
stop in the discrete + sequential + diverging palette catalogues plus the
volcano default colours, computes WCAG 2.1 SC 1.4.11 contrast (3:1)
against the chart's white background, and emits a markdown report
classifying each palette as discrete / sequential / diverging (so the
by-design pale endpoints of viridis / blues / etc. don't get flagged as
bugs). Static вЂ” pure arithmetic, no Playwright / axe-core dependency.
Informational, not a CI gate; intended as the third leg of the
"accessibility floor" alongside the two custom ESLint rules
(`plottr/no-chrome-hex-literal` and `plottr/no-css-var-in-svg`) so a
palette regression has the same chance of being caught as a chrome-color
regression.

## рџ”§ Changed

**Tool bundles are now lazy вЂ” first-visit download cuts ~60вЂ“95 % by
route.** Pre-change the SPA shipped a single ~740 KB monolith with every
tool inlined, so a mobile user opening the molarity calculator paid the
full ~1042 KB transfer cost up front. Each tool's `App` is now wrapped in
`React.lazy(() => import("..."))` in `tools/_app/tool-registry.ts`,
esbuild's `--splitting` flag emits one chunk per tool plus a shared
`_shell` chunk, and `index.html` loads the SPA entry as a module script.
A calculator-only first visit drops from ~1042 KB to ~403 KB (~61 %
smaller); a plot-tool visit fetches only that tool's chunk on navigation
behind a themed Suspense fallback. The `--chunk-names=chunks/[name]-[hash]`
flag keeps the chunk filenames stable enough for browser caching while
still cache-busting on rebuild.

## рџђ› Fixed

**Mobile topbar inside a tool now hides plot-tool jump icons again.** The
`@media (max-width: 900px)` rule in `index.html` strips the in-tool
topbar to "just the two calculators + theme toggle" via `[data-back]` /
`[data-tool="power"]` / `[data-tool="molarity"]` selectors вЂ” but the SPA
migration of the topbar into a React component (`tools/_app/App.tsx`)
dropped those attributes, so the selector matched nothing and every tool
icon stayed visible on phones (an invitation to open a plot tool that
won't render usefully at that width вЂ” recall the desktop-first 1100 px
floor for plot tools). Restored by passing `extraAttrs` through the
`IconButton` factory so the React variant emits the same `data-*` hooks
the CSS expects.

**Stuck chunk-loading spinner now self-recovers.** A flaky network or a
backgrounded tab could leave a tool's `import()` promise pending forever;
React's Suspense kept showing the spinner and the only fix was a manual
page reload. Two layers of recovery: **(a)** the dynamic-import loader
retries up to 3 times with linear backoff before surfacing a rejection,
so transient CDN flakes self-heal silently; **(b)** the Suspense fallback
morphs after 6 s into a "Loading is taking longer than expected" prompt
with a Reload-page button, so genuinely hung fetches no longer trap the
user behind an infinite spinner. Both layers are scoped to the SPA's
lazy-chunk routes вЂ” the shared bundle path (loaded via `<script>` in
`<head>`) is unaffected.

**Power Analysis: question banner now spans the full row beneath the
test / solve-for pickers.** The banner's flex sizing was lost in the
iframeв†’SPA migration along with the deleted `power.html` (its layout
CSS lived there); the tile rendered at content width and either squeezed
in beside the pickers or wrapped to a narrow second-line fragment.
Restored the original "full-width below" layout via `flex: 1 1 100%` on
the banner.

**Power Analysis: tile-to-tile spacing now matches the rest of the app.**
Power's top row and main row used `gap: 20`; every other tool (plot-
scaffold sidebars, molarity, shared upload panels) uses `gap: 10`, so
the calculator visibly drifted apart from the canonical chrome. Brought
both gaps down to 10.

**`prefers-reduced-motion` was only partially honoured.**
`PlotToolShell.tsx`'s step-change opacity fade already checked the media
query and skipped the fade for users who set "reduce motion" in their
OS вЂ” but the chrome's CSS-side transitions (button hovers, segmented
controls, disclosure-arrow rotation, landing-page tile scale,
prefetch-bar width slide, theme/language toggle hovers вЂ” nine transition
declarations across `tools/components.css` + `index.html`) ran
unconditionally. Added a universal `@media (prefers-reduced-motion:
reduce)` block in `components.css` that zeros `animation-duration` /
`transition-duration` site-wide; the existing JS check in
`PlotToolShell` is unchanged.

**Volcano "not significant" default colour was below WCAG 3:1.**
`VOLCANO_DEFAULT_COLORS.ns` was `#999999` (2.85:1 against white),
failing WCAG 2.1 SC 1.4.11 by 0.15. Bumped to `#737373` (3.27:1) вЂ” same
neutral character, a hair darker, comfortably over the bar. NS dots
typically dominate the chart (~80вЂ“95 % of points); picking the
smallest-possible step over the line is the right calibration so the
default plot still reads as "muted greys around a vivid up/down
two-colour" instead of "all three colours equally prominent". The
volcano e2e spec was updated to track the new hex.

**`tests/helpers/shared-loader.js` was silently exporting `undefined`
for `COLOR_PALETTES` and `DIVERGING_PALETTES`.** `const` bindings inside
`vm.runInContext` stay script-scoped and don't become properties of the
context object вЂ” `ctx.COLOR_PALETTES` was reading back `undefined` even
though the declarations succeeded. Switched to
`vm.runInContext("COLOR_PALETTES", ctx)` (mirroring the volcano +
discrete-palette loaders, which already had the right pattern). No
current test consumed these exports, so the bug had been latent вЂ” the
fix is preemptive for the next contributor who reaches for them.

**Heatmap and Volcano charts had no accessibility attributes.** Both
SVGs now declare `role="img"`, an `aria-label` (the user's plot title
or a default), an SVG `<title>` (renders as hover tooltip on supporting
browsers), and an SVG `<desc>` that screen readers read out вЂ” for
Heatmap, the row Г— column dimensions and whether clustering is enabled;
for Volcano, the up / down / not-significant counts via the existing
`summarize()` helper. Brings them up to the same baseline the six other
charts already had. Aequorin's inset bar plot also gained the `<desc>`
it was missing.

**Per-element a11y enrichment for the four sparse charts.** Lineplot
trace paths gain `aria-label="${name}: ${N} x-points"`; Aequorin
timecourse traces gain `aria-label="Trace: ${prefix}"`; UpSet
intersection bars gain `aria-label="${intersectionLabel}: ${size}
elements"` (using the existing `intersectionLabel` helper); Scatter's
regression-line group gains `aria-label="Linear regression: slope X,
intercept Y, RВІ Z, n=N"` when the regression overlay is active. Boxplot
already had per-rect labels; this round closes the four gaps to match.
