# Plöttr v0.9.1 — release notes

_Released 2026-04-21._

A pure refactor / test-coverage release. The shared sidebar wrapper lifts
into `_shell/PlotSidebar.tsx`, Heatmap and Aequorin (the two largest single-
file `.tsx`s) get folder-split into cohesion-keyed sibling modules, and four
plot tools that had only fuzz-coverage on their pure helpers get dedicated
unit-test suites — bumping the deterministic test count 676 → 750.

## Changed

**Internal — shared `_shell/PlotSidebar.tsx` component replaces seven copies of the sidebar frame.** Every plot tool (upset, scatter, lineplot, venn, boxplot, aequorin, heatmap) rendered its controls column inside an identical `<div style={{ width: 279, flexShrink: 0, position: "sticky", top: 24, maxHeight: "calc(100vh - 90px)", overflowY: "auto", display: "flex", flexDirection: "column", gap: 10 }}>` wrapper — modulo heatmap which used `width: 280` and opted out of sticky positioning. Extracted the frame into `tools/_shell/PlotSidebar.tsx` with a `sticky?: boolean` (default true) and `width?: number` (default 279) prop so the heatmap case is expressible without duplicating the whole style block. Migrated all seven call sites; no behavioural change (heatmap keeps `sticky={false} width={280}`, everyone else uses the defaults which reproduce the prior inline styles byte-for-byte). Clears item #2 from the architectural-follow-ups shortlist.

**Internal — Heatmap `.tsx` folder-split into six sibling modules.** `tools/heatmap.tsx` had grown to 2817 lines — the last of the "three biggest tools" the architectural review flagged for folder-splitting, after boxplot (done) and aequorin (done). Split into `tools/heatmap/` following the same cohesion-keyed pattern: five new sibling modules plus the existing `helpers.ts`. `chart.tsx` (769 lines — `PaletteStrip` preview tile + `HeatmapChart` forwardRef with all dendrogram / cluster-strip / selection-mask / tooltip logic and the memoised cells array), `controls.tsx` (433 lines — `ClusterModeControl` + `ControlSection` collapsible wrapper + the full `PlotControls` sidebar panel, which pulls the `PaletteStrip` preview from `./chart` so palette picker + colourbar stay in sync), `plot-area.tsx` (251 lines — `DetailView` zoom tile with its pruned-subtree dendrogram logic + `DetailPreviewCard` CSV preview + `DETAIL_DENDRO_STROKE_WIDTHS` preset), `reports.ts` (218 lines — pure `buildHeatmapRScript` pheatmap script builder + `buildCsvExport` headers/rows pair, no React dependency), `steps.tsx` (103 lines — `UploadStep` with the "How to use" info card). `tools/heatmap/index.tsx` (526 lines) is the new esbuild entry and holds the `VIS_INIT_HEATMAP` constant, the deterministic `EXAMPLE_CSV` dataset (500 genes × 6 samples across five latent response patterns), and the `App` orchestrator — state wiring, clustering memos, selection handlers, parse pipeline, reset flow. Each sibling stays under 800 lines. esbuild entries swapped `tools/heatmap.tsx` → `tools/heatmap/index.tsx` (output `tools/heatmap/index.js`); `tools/heatmap.html`, `eslint.config.js`, and `.prettierignore` updated to match. `tests/helpers/heatmap-loader.js` and `tests/fuzz/heatmap.fuzz.js` need no changes: the fuzz harness already consumed pure helpers via `tools/heatmap/helpers.ts`, which stays put and is now imported by the four new sibling modules. All 750 tests, `fuzz:heatmap` (1000/1000), lint, typecheck, and format:check pass unchanged; `npm run build` produces `tools/heatmap/index.js` at 62.1 KB (previously `tools/heatmap.js` ~62 KB). No user-visible change — auto-prefs key (`"heatmap"`), UI, chart output, and exported SVG/PNG/CSV/R-script content are identical. Completes the per-tool folder-split for the three biggest tools identified in the architectural review.

**Internal — Aequorin `.tsx` folder-split into seven sibling modules.** `tools/aequorin.tsx` had grown to 4172 lines — the second-largest tool file in the repo after the recently-split boxplot. Split into `tools/aequorin/` following the same pattern: six new sibling modules plus the existing `helpers.ts`. `chart.tsx` (734 lines — `Chart` forwardRef + `InsetBarplot` forwardRef + memoised `FacetChartItem`, pure SVG renderers), `stats-panel.tsx` (641 lines — `AequorinStatsDetail` + `AequorinStatsPanel`, wired to the shared `_shell/stats-dispatch`), `reports.ts` (215 lines — pure string-builders + test/post-hoc label dictionaries + `AQ_ERROR_BAR_LABELS` + `computeAqAnnotationSpec` / `computeAqSummaryText` / `buildAqSetTextBlock` / `buildAqAggregateReport` / `buildAqAggregateRScript`, no React dependency), `plot-area.tsx` (828 lines — the `PlotPanel` forwardRef orchestrator that owns the collapsible time-course chart tile, the collapsible Σ barplot tile with its per-replicate CSV table, and the stats-panel embed, plus the sidebar `ConditionEditor` and the sticky `SampleSelectionOverlay`), `steps.tsx` (545 lines — `HowToSection` / `UploadStep` / `ConfigureStep`), `controls.tsx` (585 lines — `ControlSection` + `SubHeading` primitives + the full `PlotControls` panel, which imports `ConditionEditor` from `plot-area`). `tools/aequorin/index.tsx` (706 lines) is the new esbuild entry and holds only the `VIS_INIT_AEQUORIN` constant + `App` orchestrator — state wiring, calibration memos, condition handlers, parse pipeline, reset flow, download handlers. Each sibling stays under 1200 lines. esbuild entry swapped `tools/aequorin.tsx` → `tools/aequorin/index.tsx` (output `tools/aequorin/index.js`); `tools/aequorin.html`, `eslint.config.js`, `.prettierignore`, and the stale comment in `tools/aequorin/helpers.ts` + `tests/fuzz/aequorin.fuzz.js` updated to match. `tests/helpers/render-loader.js` already had the flat-to-folder fallback from the boxplot split, so no test-loader changes were needed. The pre-commit hook's `tools/*/index.js` glob already covered the new output path. All 750 tests and `fuzz:aequorin` (1000/1000) pass unchanged; `npm run build` produces `tools/aequorin/index.js` at 88.8 KB (previously `tools/aequorin.js` ~88 KB); `typecheck` / `format:check` green. No user-visible change — auto-prefs key (`"aequorin"`), UI, chart output, and exported SVG/PNG/CSV/TXT/R-script content are identical. Completes the per-tool folder-split for the three biggest tools identified in the architectural review: boxplot (done), aequorin (done), heatmap (remaining).

## Added

**Internal — dedicated unit-test files for scatter, heatmap, aequorin, and lineplot pure helpers.** Follow-up to the v2.3.0 code-review: four of the seven plot tools had their `helpers.ts` modules covered only by fuzz harnesses (which assert structural invariants, not numerical correctness). A silent sign-flip or off-by-one in any of the calibration formulas, normalisation paths, regression math, or per-x stats pipeline could have passed fuzz unchanged. Added `tests/scatter.test.js` (13 tests — `fmtTick` formatting boundaries, `SHAPES` catalogue, `computeLinearRegression` with perfect / noisy / degenerate / null-filtered fixtures and the "zero y-variance → r² = NaN" edge), `tests/heatmap.test.js` (26 tests — `finiteMean` / `finiteSD` with NaN handling, `normalizeMatrix` for all three modes including zero-variance row and `log2` guard, `autoRange` symmetric / single-value / all-NaN branches, `buildDendroLayout` + `pruneDendroTree` on a hand-constructed 3-leaf tree, `fmtColorbarTick`), `tests/aequorin.test.js` (21 tests — `convertTime` round-trips, Allen & Blinks / Hill / generalized calibration pinned to textbook reference values at f=1/3, `detectConditions` pooling vs rep-wise, `smooth` null handling, `buildAreaD` / `buildLineD` path output), and `tests/lineplot.test.js` (14 tests — `buildLineD` / `formatX`, `computeSeries` group ordering + mean/SD/SEM, `computePerXStats` eligibility + BH adjustment monotonicity). Wired into `package.json` test script. Landing-page counter bumped 676 → 750.
