# Plöttr v1.0.2 — release notes

_Released 2026-05-01._

The audit-23 closeout release: Tier 1 quick fixes (cellKey collisions,
UpSet handoff guard, downloadCsv header escape, scrollable-card comment),
Tier 2 reliability fixes (column-type heuristic, scatter / lineplot prefs
persistence, wide-to-long silent-drop counts), Tier 3 prefs schema scaffold,
Tier 4 test hardening (direct distribution-primitive symmetry, ncf_sf small-d2
coverage, real R-script integration), and Tier 5 tooling robustness
(`formatP` contract, build-shared parse-before-write, pre-commit
stash-and-index-diff). A new shared `_shell/chart-annotations.tsx` collapses
three near-duplicate bracket / CLD renderers, boxplot's Facet × Subgroup gate
is dropped now that the stats panel is a single table, and a "trust badges +
how-it-works strip" lands on the landing page.

## Fixed

**Audit-23 Tier 5 — tooling robustness (3 items).** Closes Tier 5 of `audit_01-05-2026.md`. **Finding #19 — `formatP` non-finite contract pinned by tests.** The audit flagged that `formatP` collapses every non-finite p (NaN, ±Infinity, null, undefined) to `"—"` with no distinction between underflow and clearly-invalid values. The suggested "distinguish in DEBUG mode or assert in dev" fix would have broken the codebase's "silent libraries" convention (`stats.js` and `shared.js` have zero `console.*` uses). Better polish: pin the existing contract with five tests so a future deliberate change is possible but a silent drift is not. Covers null/undefined → `"—"`, NaN/±Infinity → `"—"`, negative or > 1 finite values pass through to the numeric format (trust-the-caller policy), `p === 0` (underflow) renders as `"0.0e+0"`, and the boundary thresholds (1e-3 / 1e-4 use strict `<`). **Finding #15 — `build-shared.js` parses concatenated bundle before writing.** A typo in any `tools/shared-*.js` (missing brace, stray backtick, mistyped regex) would otherwise atomically overwrite `tools/shared.bundle.js` with a broken file; every tool's HTML would white-screen on next reload, and in `--watch` mode the broken state persisted silently until the next save. Added `new Function(bundle)` between concat and write — parses without executing, throws on `SyntaxError`, leaves the previous good bundle untouched. Verified by injecting `const broken = {` into `shared.js`: build exits 1 with a clear error message, bundle md5 stable. **Finding #14 — pre-commit hook stash + index-relative diff.** The previous hook's `git diff` after rebuild compared working tree against HEAD, so it would silently re-stage compiled outputs that drifted from unrelated workspace state (e.g. a leftover `npm run build` no-one committed). It would also overwrite a developer's deliberate hot-patch on a compiled `.js` during live debugging. New flow: stash unstaged + untracked changes before rebuild (`git stash push --keep-index --include-untracked`), diff the rebuilt outputs against the INDEX (not HEAD), re-stage only those, then pop the stash on EXIT trap. If pop conflicts (because a hot-patch on a compiled output collides with the rebuild), leave the stash in place with a clear warning so the user can resolve manually. Total deterministic tests 886 → 891.

**Scatter — visual prefs no longer wiped on every file parse.** Regression introduced by audit-23 #1 (commit `16f205f`) wasn't fully cleaned up: `doParse` in `tools/scatter.tsx` still called `setPointColor("#648FFF")` and friends after every parse, writing literal defaults back into `vis` and overwriting whatever the user had tuned. Discovered by comparing against boxplot/venn behaviour (which only wipe the per-group colour dicts, not global style slots). Fixed by removing the visual-pref resets from `doParse` — only the column-index slots (`colorMapCol` / `sizeMapCol` / `shapeMapCol`) and `filterState` reset on parse now, since those genuinely tie to the previous dataset's column layout. Per-category mapping dicts (`colorMapDiscrete` etc.) ALSO persist across parses by design: keyed by category name, so orphaned keys from old datasets are harmless and matching names retain the user's colour. Drops the unused `setRegression` alias added in `16f205f`.

**Audit-23 Tier 2 reliability fixes (4 items).** Closes Tier 2 of `audit_01-05-2026.md`. **#9 — `guessColumnType` mis-classified year and binary columns.** A `Year` column (`2024 / 2025 / 2026`) was 100% numeric so the heuristic returned `value`, landing it on a continuous axis instead of grouping by year. Same for binary `0/1` flags. New pre-check before the value branch returns: ≤ 12 distinct values, distinct count < 30% of row count, AND every value is an integer ⇒ demote to `group`. Catches years, binary flags, and small integer enumerations without affecting genuine continuous small-N data. **#1 — Scatter + lineplot styling now persists across reloads.** Scatter had 12 plot-state slots in local `useState` (point colour/size/opacity/stroke, `colorMapPalette`, three discrete-mapping dicts, `sizeMapMin/Max`, `refLines`, full `regression` sub-object) that did not round-trip through `PrefsPanel` or `auto-prefs` localStorage. Lineplot had two (`errorType`, `showStars`). All migrated into `VIS_INIT_*` mirroring the boxplot pattern from commit `0b7fecb`. Per-category mapping dicts are keyed by category NAME so they survive a dataset swap when names match. Column-index slots stay local — they're tied to the dataset, not a visual pref. **#10 — `wideToLong` silent cell drops surfaced.** The wide-to-long reshape silently dropped empty / non-numeric cells with no count or banner. `wideToLong` now returns `{headers, rows, skipped}`; boxplot surfaces a one-line yellow banner above the plot when `skipped > 0` so the user knows their sample size shrank. **#17 — `reshapeWide` empty-group merges surfaced.** Multiple rows with empty group cells silently collapsed into a single `"?"` bucket. `reshapeWide` now returns `{headers, rows, unlabelled}`; the OutputStep wide-CSV download tile shows a one-line italic note when `unlabelled > 0`. New tests for both helper counts plus 8 prefs round-trip tests for scatter / lineplot. Landing-page test counter 854 → 866.

**Audit-23 Tier 3 — only #8 (prefs schema-migration scaffold) landed.** `PREFS_SCHEMA_VERSION` was written at save time but never validated at load time, so the next schema bump would silently mix old + new keys via the whitelist merge. Added `migratePrefs(settings, fromVersion)` in `tools/shared-prefs.js`; `loadAutoPrefs` threads the parsed blob through it. Today every version is 1, so the function is pass-through; the next bump adds a `case 1: settings = migrate1To2(settings); v = 2;` step. Future versions (> current) return null so the load falls back to defaults rather than risking a mixed-shape merge. Four tests pin the contract (current-version pass-through, missing-version back-compat, future-version reject, loadAutoPrefs fallback). **#3 (CSP) and #4 (iframe sandbox) attempted and reverted.** Both shipped briefly and were caught in browser smoke-test: a `<meta http-equiv>` CSP that locks `default-src` blocks Plöttr's `<a href="blob:" download>` programmatic-click downloads even with `blob:` whitelisted in `default-src` + `connect-src`; the iframe sandbox `allow-downloads` token has the same fragility under Chromium with programmatic anchor clicks. Plöttr's whole download path uses `URL.createObjectURL` + JS-synthesised anchors, so both protections are incompatible with the current UX. Deferred — revisit when downloads switch to user-clicked anchors rendered into the DOM (which would re-enable both), or when GitHub Pages gains HTTP-header CSP support, or when `<iframe csp=…>` per-frame policy gains broader browser support. Landing-page test counter 866 → 870.

**Audit-23 Tier 1 quick fixes (4 items).** Closes the four highest-regret / lowest-effort items from `audit_01-05-2026.md`. **#2 — Boxplot cellKey collision + stale-key cleanup.** `cellKey(facetCat, sgName)` switched from `${facetCat}::${sgName}` to `JSON.stringify([facetCat, sgName])` — a category named `"S3::pGFP"` paired with `"WT"` previously collided with `"S3"` paired with `"pGFP::WT"` (plant-science labels routinely contain `::`). Also: `setRenameVal` and `updateRole` now dispatch `clearCells` when they touch the active facet or subgroup column, so renaming a facet category mid-session no longer orphans `cellAnnotations` / `cellSummaries` entries under the old name. **#5 — UpSet handoff size guard.** The Venn → UpSet handoff (sessionStorage + postMessage) now rejects payloads larger than `FILE_LIMIT_BYTES` (2 MB) before reaching the parser, and sanitises `payload.fileName` (max 255 chars, path-separator strip, leading-dot strip) so a crafted handoff can't produce a download name like `../../etc/passwd`. **#18 — `_shell/ScrollablePlotCard.tsx` header comment** updated to match reality (only UpSet uses it; venn/heatmap don't need the affordance). **#16 — `downloadCsv` headers were unquoted.** A header containing a comma (`"Sample, Note"`) wrote a malformed first line that Plöttr's own `parseRaw` split into N+1 columns on re-import. Headers now go through the same RFC 4180 escape as data cells. Pure CSV-string assembly extracted as `buildCsvString(headers, rows)` so it can round-trip through `parseRaw` in tests without DOM/Blob. New regression suite in `tests/parsing.test.js` (4 tests: plain round-trip, comma-in-header, quote-in-cell, newline-in-cell). Landing-page test counter 850 → 854.

## Added

**Audit-23 Tier 4 — test coverage hardening (3 items).** Closes Tier 4 of `audit_01-05-2026.md`. **Finding #12 — direct distribution primitive symmetry / boundary tests** in `tests/stats.test.js`. The primitives (`normcdf`, `tcdf`, `fcdf`, `chi2cdf`, `norminv`, `tinv`) were tested only transitively through high-level callers, masking regressions that would re-derive or clamp before surfacing. Adds direct tests: `normcdf` reflection symmetry across x; saturation at extreme ±x; `norminv` boundary contract (returns ±Infinity at p=0/1, no NaN); `tcdf(-x, df) = 1 - tcdf(x, df)` symmetry; `tcdf(0, df) = 0.5`; `fcdf(0, ...) = 0` and out-of-domain saturation; `chi2cdf(x ≤ 0, k) = 0`; `chi2cdf` monotonicity in x; `tinv ↔ tcdf` round-trip at boundary df ∈ {1, 2, 4, 50, 200}; `fcdf + fcdf_upper = 1` in the central body. **Finding #11 — `ncf_sf` small-d2 coverage** in `tests/power.test.js`. The closed-form normal-approximation short-circuit (`tools/stats.js:490`) fires only when `d2 > 4` (the variance formula has `(d2-4)` in the denominator). For `d2 ∈ {1,2,3,4}` the code falls through to the Poisson-mixture loop, and the audit-22 #13 widening made the loop wider but never tested that regime. Adds five tests: bounded output `[0,1]` across λ; monotone-in-f at fixed (d1, d2, λ); monotone-in-λ at fixed (d1, d2, f); central-F anchor at λ=0; far-tail sanity at small d2 + huge λ. Total deterministic tests 870 → 886. **Finding #6 — R-export integration test** as `benchmark/run-r-export.js`. The Plöttr stats panel's "⬇ R" button generates a script the user re-runs in RStudio for reproducibility. The existing `tests/r-export.test.js` checks the script is _parseable_ (formatting, escaping); nothing executed it in real R until now. The new harness picks a fixed two-group dataset, runs `tTest` in JS, builds the same R script the user would click via `buildRScript`, executes via `Rscript --vanilla`, parses the t-test block's `p-value = X` line, and asserts JS p ≈ R p within 1e-3 absolute / 1e-2 relative. Initial run reports JS p = 0.08051624 vs R p = 0.08052000 (Δ = 3.76e-6). Gated behind `npm run benchmark`; `Rscript` not on PATH ⇒ skip note + exit 0 (CI without R still passes). The reproducibility claim is now actually verified end-to-end.

**Landing page — trust badges near the tagline + a "How it works" step strip under the tile grid.** The `846 internal tests / 293 cross-checks vs R 4.5` line at the foot of the page was Plöttr's strongest trust signal for a scientific audience but the least visible piece of chrome on the landing page. Promoted that signal into a pill-shaped `Validated against R 4.5` badge (links to `benchmark.html`, same destination as the footer count) sitting directly under the tagline, paired with a `Data stays in your browser` badge for the browser-only / no-tracking story — two trust pills, low-saturation, readable in a glance. Added a second new element below the tile grid: a `How it works` step strip — four numbered pills (`Paste CSV → Assign roles → Tweak plot → Download SVG + R`) reflecting the common Upload → Configure → Filter → Output stepper every plot tool shares, so first-time visitors know the flow is consistent across tools before they click into one. Both elements are desktop-only: the mobile media rule (`max-width: 900px`) hides `.how-it-works` alongside the existing plot-tile hides (the four-step flow only applies to the plot tools, which don't render on phones). The trust badges stay visible on mobile since they describe the product itself, not the plot workflow. No JS wiring beyond the existing `data-benchmark-link` handler already used by the footer link.

## Changed

**Significance brackets + CLD-letter labels lifted into a shared `_shell/chart-annotations.tsx` module.** Three near-identical bracket renderers (boxplot vertical, boxplot horizontal, aequorin vertical) and three near-identical CLD-letter renderers were collapsed into two parameterised components — `<SignificanceBrackets>` (orientations: `vertical-top`, `horizontal-right`) and `<CldLabels>`. The components are pure presentational: the caller passes pre-leveled pairs (`assignBracketLevels` upstream) and pre-computed labels (`pStars(p)` upstream), and the component only translates `(i, j, _level)` → SVG geometry via an `axisCoord(i)` callback. Boxplot's styling is the source of truth: stroke `#333`, bracket fontSize 12, CLD fontSize 13, text colour `#222`. **Aequorin's brackets and CLD letters now adopt this style** (previously stroke `#222`, fontSize 10) — visible bump in the inset bar chart so significance markers carry the same visual weight as in boxplot. Aequorin's chart no longer recomputes the label inline (the spec already carried `pr.label`). Closes audit M7. Both tools' fuzz harnesses pass 1000 iterations clean post-refactor; visual regressions are not crash-detectable so verify by eye on a calibration dataset before release.

**Boxplot — Facet by + Subgroup by are no longer mutually exclusive.** When the stats panel was a stack of `StatsTile` widgets the two splitters together would have exploded vertical real estate, so the original implementation hard-blocked the combination by clearing one whenever the other was set. Now that the stats panel is a single lean table at the bottom, the constraint is unnecessary and the four valid modes — flat / facet only / subgroup only / facet × subgroup — are all selectable. **Data layer**: the per-mode `facetedData` and `subgroupedData` builders were unified through a shared `buildCellGroups` helper. `facetedData` carries optional `subgroups` + `flatGroups` per facet; `subgroupedData` is computed only in the no-facet path. **Stats reducer**: collapsed the three slot pairs (`flat*` / `facet*` / `subgroup*`) into a single `cellAnnotations` / `cellSummaries` dict keyed by composite `${facet}::${subgroup}` strings (with empty strings for missing dimensions and a `flat` sentinel for the singleton). One `clearCells` action replaces the previous per-mode clear actions. **Layout**: a single bottom `BoxplotStatsPanel` in every mode, with explicit per-tile naming so users can read which scope each row covers — `Facet × Subgroup` setLabel and `${facet} — ${subgroup}` row name when both are active. The `setLabel` is `Facet`, `Subgroup`, or empty in the simpler modes (consistent with the prior single-mode panels). **Chart layer**: `FacetTrio` no longer hard-codes `subgroups: null`; it forwards each facet's `{subgroups, flatGroups}` into `BoxplotChart`. Per-facet annotation merging reuses the existing `mergeSubgroupAnnotations` helper, called once per facet with that facet's subset of `cellAnnotations`. **Controls**: the Split-by dropdowns now filter each other's pool so the same column can't drive both axes (degenerate — every facet would contain a single subgroup), and selecting one no longer clears the other. **Tests**: stats-reducer tests rewritten for the cellAnnotations/cellSummaries shape; new `tests/boxplot-helpers.test.js` covers the per-facet `mergeSubgroupAnnotations` extraction pattern (key isolation across facets, subgroup-only mode using empty facet key, returning null when no specs are present). Landing-page test counter bumped from 846 → 850.
