# Plöttr v1.6.0 — release notes

_Released 2026-05-16._

This release is the outcome of a pre-launch correctness audit of the
statistics kernel and the math→plot rendering layer, plus one heatmap
usability feature. The audit confirmed the kernel's formulas are sound —
nothing here changes a result on well-formed data. What it fixes is a set
of edge cases that previously produced a silently-wrong answer or a
misleading chart.

## 🔧 Changed

**The heatmap colorbar auto-ranges when you cross between a diverging and
a sequential palette.** Diverging palettes (RdBu, bwr, RdYlBu, spectral)
read correctly only when vmin/vmax are symmetric about 0; sequential
palettes (viridis, …) want the data's own min/max. Selecting a palette of
the other kind now refills vmin/vmax accordingly — symmetric for
diverging, data-fit for sequential. The trigger is keyed on the palette's
diverging-ness, so a sequential→sequential swap (viridis→plasma) leaves a
manually-entered range untouched. Companion to the normalization-driven
re-range below.

## 🐛 Fixed

### Statistics — degenerate-input handling

**Mann-Whitney U on all-identical data returns an error, not a false
"p = 1".** When every observation across both groups is identical the
tie-corrected variance collapses to exactly 0 and the test is undefined.
The function previously reported `z = 0, p = 1` — a confident "not
significant" — with no error. It now returns an error, matching the guard
Kruskal-Wallis already had.

**Games-Howell and Dunn's test reject degenerate input instead of
emitting a silent NaN.** Games-Howell now errors when any group has fewer
than 2 observations (a singleton has no sample variance, which slipped
past the zero-variance check as `NaN`); its Welch-Satterthwaite df is
also clamped to ≥ 1 against floating-point rounding. Dunn's test errors
on too-few observations or an empty group, mirroring the Kruskal-Wallis
precondition — Dunn is its post-hoc.

**`selectTest` reports a non-finite diagnostic honestly.** A group
carrying ±Infinity made Shapiro-Wilk / Levene return `NaN`; the
recommendation narrative then asserted "non-normal" / "unequal variances"
from that garbage (`NaN >= α` evaluates to `false`). Such a diagnostic is
now reported as "unknown". The recommended test was, and remains, Welch
regardless.

**Distribution functions reject an invalid degrees-of-freedom argument.**
`tcdf`, `tcdf_upper`, `tinv`, `fcdf`, `fcdf_upper`, `chi2cdf`, `chi2pdf`,
and the noncentral `nctcdf` / `ncf_sf` / `ncchi2cdf` returned a finite
but wrong number for a df ≤ 0 (`tcdf(t, 0)` returned 1). They now return
`NaN`, consistent with the guard `chi2inv` already had.

### Math → plot rendering

**Heatmap colorbar re-ranges when you switch the normalization mode.**
Switching to Z-score (values ≈ ±3) or log₂ shifts the value scale
entirely, but vmin/vmax stayed on the raw-data range — rendering the
heatmap as a saturated, near-uniform block until "Auto from data" was
clicked by hand. It now re-ranges automatically on a normalization
change.

**Heatmap dendrograms no longer draw inverted (crossed) branches.** When
a cluster had to be merged with no distance information — all-NaN
distances, e.g. rows whose finite values don't overlap — the merge node
was placed at height 0, below its own children, drawing a crossed
dendrogram. Merge heights are now clamped to stay monotonically
non-decreasing; the normal single/complete/UPGMA path is already
monotone, so the clamp is a no-op there.

**Aequorin's auto Y-range no longer clips the mean ± SD ribbon.** The
range was computed from the raw replicate cells, but `mean + SD` can
exceed the largest single replicate — so the top of the error band was
flattened against the plot frame, reading as smaller than it actually is.
It now ranges over the ribbon edges the chart draws.

**UpSet's multi-set p-value is exactly 0 for an impossible
intersection.** The Poisson-approximation path (used for large set
families) returned a small non-zero tail probability when the observed
intersection size exceeded the smallest set — an impossibility. It now
returns 0, matching the exact path.

## 🔬 Verified, not changed

The audit also flagged the compact-letter-display algorithm as a possible
correctness gap on intransitive significance patterns. Tracing the
split-and-absorb algorithm showed it provably preserves both CLD
invariants — significantly-different means never share a letter,
non-significantly-different means always do — so no code changed; the
intransitive cases were instead pinned with regression tests.
