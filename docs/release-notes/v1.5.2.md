# Plöttr v1.5.2 — release notes

_Released 2026-05-16._

A one-fix patch on top of v1.5.1: long boxplot category labels no longer
clip off the left edge of the SVG when the x-axis labels are rotated.

The rest of the cycle was test-suite hardening and type-safety work with
no behaviour change. The `## 🔬 Internal` section below records it so the
release history is complete.

## 🐛 Fixed

**Long boxplot category labels no longer clip off the left edge when the
x-axis labels are rotated.** When x-axis tick labels are rotated they are
anchored at their tick and tilt down-left, so a long label on the
left-most group extends to the left of its tick. The chart's left margin
was hard-coded to a 62 px floor — sized only for the y-axis title and
numeric tick labels — so that overhang ran off the left edge of the SVG,
and off the edge of the exported file. `computeChartMargins` now reserves
the rotated label's horizontal extent (`labelWidth × cos(angle)`) in the
left margin, mirroring the vertical reservation the bottom margin already
made for the same rotated labels. Short labels, and rotation-free charts,
keep the 62 px floor untouched.

## 🔬 Internal

_No user-visible effect. This section records the verification work done
during the cycle so the release history is complete._

### Statistics kernel re-verified with mutation testing

All six modules of the `tools/_core/stats/*` numerical kernel were swept
with Stryker mutation testing — which mutates the source (flips
comparators, swaps constants, deletes statements) and re-runs the suite
against each mutant, measuring whether the tests actually _catch_ bugs
rather than merely pass. Final mutation scores:

| Module       | Score                          |
| ------------ | ------------------------------ |
| `format.ts`  | 95.56 % (100 % non-equivalent) |
| `cluster.ts` | 87.97 %                        |
| `tests.ts`   | 87.83 % raw / 88.42 % covered  |
| `posthoc.ts` | 79.80 % raw / 81.20 % covered  |
| `msi.ts`     | 72.20 %                        |
| `dist.ts`    | 69.12 % raw / 69.93 % covered  |

New boundary and uncovered-path tests were pinned across all six — the
k-means multi-iteration loop and empty-cluster reseed, the `gammaln`
reflection, `norminv` domain guards and one-tailed power branches in
`dist.ts`, the post-hoc `selectTest` routing and Shapiro-Wilk narrative
branches, correlation error-message content and the Spearman Fisher-z
CI, and more. After the sweep every residual survivor is either provably
_equivalent_ (a syntactically-different but semantically-identical
mutation that no test can distinguish) or only killable by contrived
hostile inputs. The lower scores on `dist.ts` / `msi.ts` are intrinsic
ceilings — those files are almost entirely iterative numerics whose
internal steps converge to the same R/SciPy-cross-validated output
regardless of mutation.

A latent Stryker harness bug was found and fixed along the way: under
Stryker's per-mutant isolation the test loaders spawned one esbuild
service per mutant and exhausted the OS process table mid-run, silently
scoring later mutants as false survivors. Loaders now memoize their
bundles on disk. Each target also gets its own mutation report instead
of overwriting a shared file, and genuinely-equivalent lines carry an
`equiv-mutant` tag. The full per-file breakdown lives in
`tests/CLAUDE.md`.

### React tier is now `any`-free end to end

The React/TSX surface carries no `: any` annotations, no `any` inside
`useState` / `useRef` generics or `Record` / `Map` value types, no
`as any` / `as unknown as` escape-hatch casts, and the repo holds no
`@ts-ignore` / `@ts-expect-error` / `@ts-nocheck` directives anywhere.
Prop bags and component state that were previously loosely typed are now
backed by explicit `interface` declarations. This is purely a
maintainability change — the compiled output and runtime behaviour are
unchanged — but it means a future type error in the React tier is now a
hard `tsc` failure rather than something an `any` silently swallows.

### SVG-snapshot visual-regression suite

A new test suite renders each chart tool to SVG and snapshots the
output, so an unintended change to chart geometry — margins, tick
placement, element ordering — now surfaces as a failing snapshot diff
instead of slipping through. Edge-case snapshots for scatter, aequorin,
and lineplot were added on top. The deterministic suite is now 38 files
/ 1,734 tests.
