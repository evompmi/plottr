# Plöttr v1.5.1 — release notes

_Released 2026-05-14._

A one-fix patch on top of v1.5.0: the seeded PRNG behind boxplot jitter
no longer pins single-point-per-category color-by sources to the left
edge of their jitter band.

## 🐛 Fixed

**Boxplot color-by jitter now spreads single-point categories.** The
Park-Miller LCG behind every `seededRandom(seed)` call returns roughly
`seed × 16807 / 2³¹` on its first call — so adjacent small seeds (the
per-source `gi*1000 + si*100 + 42` pattern boxplot jitter uses) all
produce near-identical first values. A color-by source with only one
point per category therefore had every dot pinned to the left edge of
the jitter band, even though the band itself was correctly sized.
`seededRandom()` now burns one LCG step before returning so the first
observable output already lives in the well-mixed region of the cycle.
Determinism is preserved — same seed still gives the same sequence; the
sequence just starts one step further along.
