# Plöttr v1.0.0 — release notes

_Released 2026-04-22._

First release under the Plöttr name. Supersedes the `dataviz` lineage
(v0.4.0 – v0.10.0). No code regressions — the `1.0.0` bump marks the rename
and stable public surface, plus a small UpSet feature pass (set-include /
nudge / handoff polish, label toggles), the moss accent for "done" /
"significant" signals, and the new horizontal stepper.

## Changed

**Project renamed from `dataviz` to `Plöttr`.** "dataviz" is a generic term that says nothing about the project — it's hard to search for, hard to reference in talks, and collides with every other plotting helper on GitHub. Renamed to **Plöttr** (lowkey Nordic-glyph joke, pronounceable once spoken). Updated `package.json` name, `README.md` header / live-URL / source-URL / clone-URL / citation block (with a transition note pointing back to the dataviz repo), landing `index.html` `<title>` and `<h1>`, every tool HTML `<title>` (`aequorin.html`, `boxplot.html`, `heatmap.html`, `lineplot.html`, `molarity.html`, `power.html`, `scatter.html`, `upset.html`, `venn.html`), and `benchmark.html` `<title>`. The live site will move to `evompmi.github.io/plottr` and source to `github.com/evompmi/plottr`; the old `dataviz` repo will be left in read-only mode. **Intentionally unchanged:** every `localStorage` key and cross-tab event name (`dataviz-theme`, `dataviz-theme-change`, `dataviz-handoff`, per-tool `dataviz-prefs:*` keys). Renaming them would invalidate every existing user's saved theme choice and per-tool auto-prefs on first visit to the new domain — not worth the pain for an internal name. If we ever do migrate, it should be a one-shot read-old-write-new shim on load, not a hard break.

**UpSet Configure step — intersection-cutoff Min/Max use the canonical `−`/`+` stepper.** The two degree inputs were rendered as bare `<input type="number" className="dv-input">`, which means the browser's native stacked up/down spinner buttons appeared on the right edge. Every other numeric input in the app uses the shared `NumberInput` component (already declared as a global in `types/globals.d.ts`), which wraps the input with `−` and `+` buttons sharing the `.dv-num`/`.dv-num-btn` chrome and hides the native spinner via `appearance: textfield` + `::-webkit-inner-spin-button { -webkit-appearance: none }`. Swapped both inputs in `tools/upset.tsx` for `NumberInput` (same `min`/`max`/`step`/`value`/`onChange` contract; bumped width 72 → 96 to fit the buttons). The clamping logic that keeps `min ≤ max` is unchanged.

**Venn → UpSet handoff now preserves the long-format signal.** When Venn's upload parser pivots a 2-column `(item, set)` file via `parseLongFormatSets`, the handoff payload sent to UpSet was still hardcoded as `format: "wide"` — so UpSet's `handleHandoff` ran `parseSetData` on the 2 column headers and reproduced the same "2 sets" misread that originally triggered the nudge. Threaded an `isLongFormat` flag through `App` → `ConfigureStep`, set it in `doParse` alongside `usedLongFormat`, and swapped the handoff's `format` field to `isLongFormat ? "long" : "wide"`. UpSet's existing `payload.format === "long"` branch already routes to `parseLongFormatSets`, so no UpSet-side changes were needed.

**UpSet Configure step — "Sets to include" checkboxes themed to the CTA accent.** The set-include picker checkboxes used the browser-default accent (blue on most platforms) while the rest of the app's checkboxes — filter panels, group pickers, stats-panel toggles, and the paired Venn set picker — render with `accentColor: "var(--cta-primary-bg)"`. Added the same style to the `<input type="checkbox">` in `tools/upset.tsx` so the chrome color is consistent across tools.

**Venn upload — long-format CSVs are now detected and pivoted instead of silently parsed as 2-set wide.** A 2-column file `(item, set)` with many distinct set labels in column 2 previously went through `parseSetData`, which treats each column header as a set — so a 10-set long-format file was read as a "2-set" Venn (the two column names), and the 4+-sets UpSet nudge never fired because `allColumnNames.length` was always 2. Added a detection pass in `doParse` (`tools/venn.tsx`): when `headers.length === 2` and column 2 has ≥ 4 non-empty cells with at least one repeated label (`distinct < total`), pivot the rows via `parseLongFormatSets` from `shared.js` before running the set-picker / nudge logic. The repeat test is the key signal — a genuine 2-column wide Venn (`Set A` / `Set B`) has entirely unique items per column, while any long format has repeated group labels. The pivoted result flows through `allColumnNames` / `allColumnSets` unchanged, so downstream code (set picker, intersection math, export) works without further branching; the `>= 4` nudge check now correctly fires on a 4+-set long-format upload.

**Venn upload step — UpSet nudge is now persistent, promoted to a pill CTA, set-picker checkboxes themed.** The 4+-sets banner previously read "UpSet plots read better than Venn diagrams above 3 sets" with the "Open in UpSet tool →" rendered as a plain inline link next to the 💡 icon — the hard constraint (Venn only renders 2–3 sets) was implicit and the link was easy to miss. Tightened the copy to "Venn diagrams only render 2 or 3 sets. For 4+ sets, use the UpSet tool." and moved the CTA out of the sentence onto its own pill: a `--cta-primary-bg` rounded-999 chip with `--on-accent` text and the standard `--cta-primary-shadow`, so it reads as an action rather than prose. Also removed the banner's dismiss (`×`) button and the `venn-upset-nudge-dismissed` `localStorage` key that backed it — the banner now always shows whenever 4+ sets are detected, because the limitation is structural (Venn truly can't render 4+) rather than a preference, so hiding the escape hatch on repeat visits silently stranded users on an unusable tool. Also added `accentColor: "var(--cta-primary-bg)"` to the set-picker `<input type="checkbox">` — they previously used the browser default accent instead of the shared CTA chrome color that every other checkbox in the app uses (filter panels, group pickers, stats-panel toggles).

**Step nav redesigned as a classic horizontal stepper.** Replaced the pill-style uppercase step buttons (`1. UPLOAD` / `2. CONFIGURE` / …) and their `❯` chevron separators with the canonical horizontal-stepper idiom: a 36-px numbered circle per step connected by a progress line, with the step name as a label below each circle. Visual states encode progress directly on the circle — past steps show a stroke-based SVG checkmark (3-px round-capped, 18×18 path) on a filled `--step-ready` circle; the current step shows its number on the `--step-active-bg` chrome fill (keeping "you are here" distinct from "done"); reachable-but-unvisited steps show their number in a `--step-ready` outline; locked steps use a neutral outline. The connector line between circles fills `--step-ready` up to the last completed step and stays `--border-strong` gray from the current step onward, so the path fills as the user advances. Stepper sits inline in the header row as `[tool icon + title] | [stepper] | [PrefsPanel]` separated by 1-px `--border-strong` vertical bars (reuses `PageHeader`'s existing `middle` slot). Retires the chevron separators, the 2-px `box-shadow` halo, the uppercase pill treatment, and the Unicode `✓` glyph — all replaced by the richer per-state circle + SVG tick. `PlotToolShell` + `tools/shared-ui.js` changes; `tests/components.test.js` StepNavBar child-count assertion adjusted from 5 to 3 (one wrapper per step). Each step is a single focusable `<button>` containing both the circle and label, with `aria-current="step"` on the active step and `aria-label="Step N of M: Name"` for screen readers.

**Venn Configure step — set-picker tile now leads, data preview trails.** Follow-up to the upset / boxplot / aequorin reorder: when the UpSet-nudge banner appears (4+ sets detected) or the set-picker is shown (more than 3 columns), the order is now `[nudge] → [set picker] → [preview]` instead of `[nudge] → [preview] → [picker]`. The decision the user needs to make (which 2–3 sets to overlap) sits above the 8-row preview they've already seen on upload, so scrolling past the preview to find the picker is no longer required.

**Moss accent propagated from the stepper to download chips and stats greens.** After settling on moss (`#3f6b2a` light / `#7ba668` dark) for `--step-ready`, the same family now drives the two other places plot-tool chrome signals "good / significant / done": the `.dv-btn-dl` download chip and the stats-panel accents (significant p-values, achieved power ≥ 0.8, significance pairing markers, result pills). Added sibling tokens `--step-ready-bg` (`#eaf2df` / `#1c2916`) and `--step-ready-border` (`#b5cf9e` / `#3f6b2a`) to `tools/theme.css` — same pale-bg / mid-border / deep-text triplet shape as the existing `--success-*` family, but tuned to moss. Retargeted `.dv-btn-dl` in `tools/components.css` to the new triplet and swapped its hover box-shadow to moss rgba; the existing alpha + outline-darkens-on-hover behavior is unchanged. Swapped `var(--success-text)` / `var(--success-bg)` → `var(--step-ready)` / `var(--step-ready-bg)` in the stats-accent spots only: `tools/shared-stats-tile.js`, `tools/lineplot.tsx`, `tools/aequorin/stats-panel.tsx`, `tools/boxplot/stats-panel.tsx`. Danger reds (`--danger-*`) and non-stats success banners (boxplot's valid-assignment banner, venn / upset picker status, molarity calculator result tiles) are untouched — moss is reserved for "done / significant" signals, not "valid input" acknowledgements.

## Added

**UpSet plot — On/Off toggles to hide intersection-size and set-size bar labels.** Cosmetic option for posters / slides where the numbers on top of every bar are redundant with the axis or just visually noisy. Added two `Display` section toggles in the plot sidebar: "Intersection size labels" (the numbers above each vertical bar) and "Set size labels" (the numbers beside each horizontal set-size bar). Rendered as the canonical segmented On/Off button pair already used for `Show points` in boxplot (`tools/boxplot/controls.tsx`), not a native checkbox, so the affordance matches the rest of the app. Both default to On so existing users see no change. Threaded `showIntersectionLabels` / `showSetSizeLabels` through `VIS_INIT_UPSET` (so auto-prefs persist per-user) → `vis` → `UpsetChart`; the two `<g id="intersection-bar-labels">` / `<g id="set-size-bar-labels">` groups render conditionally on `!== false`. Does not affect the CSV export, R script, or any underlying data — purely what gets drawn.

## Removed

**Heatmap detail view — "Cluster n° X (rows/cols)" pill removed.** The detail card already conveys which cluster slice is being shown via the highlighted selection on the main plot (and the export filenames still carry the `_clusterN` suffix), so the redundant pill next to the Dendrogram stroke-width selector was just noise. The underlying `selection.clusterAxis` state is kept (still drives the `_cluster` filename suffix).

**HowToCard disclosure chevron enlarged.** The "How to use" collapsible card used a 14-px `›` (U+203A) glyph as its expand/collapse indicator — small, lightweight, and easy to miss against the gradient header. Replaced it with a 22×22 SVG chevron path (2.6-px round-capped stroke, `currentColor`), keeping the same 90° rotation on toggle. Now visually on par with the stepper's SVG checkmark and the rest of the chrome.

**Plot-tool page-header subtitles removed.** Every tool page previously rendered a one-line subtitle under the tool name (`"Load → label columns → filter → plot & export"`, `"XY scatter — one row per data point, one column per variable"`, `"Intersection sizes across many sets (4+ sets)"`, etc.). These strings restate what the landing-page tile already tells the user before they click in, so once inside the tool they're just noise competing with the stepper for attention. Dropped the `subtitle` rendering block from `PageHeader` in `tools/shared-ui.js` and the `subtitle` prop from `PlotToolShellProps` in `tools/_shell/PlotToolShell.tsx`. Removed per-tool `subtitle=` call-sites in `tools/lineplot.tsx`, `tools/venn.tsx`, `tools/scatter.tsx`, `tools/upset.tsx`, `tools/boxplot/index.tsx`, `tools/heatmap/index.tsx`, `tools/aequorin/index.tsx`, `tools/molarity.tsx`, and `tools/power.tsx`. HowToCard subtitles inside the collapsible upload-step info cards (contextually relevant inside the card) and the SVG chart's configurable subtitle (part of exported plot content) are untouched.
